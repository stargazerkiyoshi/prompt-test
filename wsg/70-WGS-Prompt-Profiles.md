# WGS Prompt Profiles

## 1. Prompt Profiles 的定位

WGS 的前几份文档已经定义了：

* 系统是什么
* 系统如何分层
* 模块各自负责什么
* 状态卡如何保存世界连续性
* 工作流如何运行
* 调用协议如何组织一轮调用

在此基础上，Prompt Profiles 的作用是进一步回答：

**每个模块在实际使用时，可以直接复制调用的 prompt 资产长什么样。**

因此，Prompt Profiles 不是再去定义模块职责，也不是再去解释工作流原理。
它的重点是：

* 给出每个模块的标准 prompt 骨架
* 让实际调用更直接
* 减少每轮都从零拼 prompt 的负担
* 为后续版本化优化提供基础资产

你可以把这一层理解为：

**从“协议”走向“可直接使用的 prompt 成品层”。**

---

## 2. 本文件的使用方式

本文件中的每个 Profile 都不是完全僵硬、不可调整的最终模板。
它们更适合作为：

* 初始调用模板
* 模块专用 prompt 底稿
* 后续迭代时的稳定骨架
* 你自己或其他模型继续优化的起点

在实际使用时，建议做两件事：

### 2.1 保留外层骨架

尽量保留：

* 当前模块
* 模块定位
* 本轮任务类型
* 本轮目标
* 现有世界状态
* 本轮约束
* 输出要求
* 风格要求

### 2.2 只替换任务目标和状态摘要

不同轮次变化最大的通常是：

* 本轮目标
* 当前状态摘要
* 个别附加约束
* 风格倾向

也就是说，Profile 的重点不是每次全改，而是**固定框架、局部替换**。

---

## 3. Profile 设计原则

WGS 的 Prompt Profiles 应尽量遵循以下原则。

### 3.1 一模块一骨架

每个模块都应有自己的专用 prompt profile，而不是所有模块都用完全相同的话术。

### 3.2 保持协议一致

虽然各模块有专属差异，但整体结构应尽量与 `60-WGS-Invocation-Protocol.md` 一致。

### 3.3 强调模块职责边界

每个 profile 都应清楚告诉模型：

* 这轮你负责什么
* 你不应该越权做什么

### 3.4 输出必须可回写状态卡

每个 profile 都应要求模型最终给出便于回写状态卡的摘要。

### 3.5 先求稳，再求华丽

除庄周外，大多数模块更应优先追求：

* 清晰
* 一致
* 可承接
* 可扩展

而不是追求过度文学化。

## 4. 混沌 Prompt Profile

### 4.1 适用场景

适用于：

* 初始化世界起源法则
* 补充世界运作法则或基本能量机制
* 修订法则冲突
* 提炼法则摘要

### 4.2 Prompt Profile

```text id="wgs-profile-chaos"
你现在是世界生成系统中的【混沌】。

模块定位：
你负责生成世界最底层的法则与存在边界，为整个世界提供最高层约束。
优先回答“这个世界为何成立”，再视需要补充“这个世界如何运作”。

本轮任务类型：
【初始化 / 扩展 / 修订 / 校验 / 摘要更新】

本轮目标：
【例如：为当前世界生成 4—7 条最基础、最通用、可长期共存的起源法则；如有必要，再补充原初能量形式、超凡边界或一条隐藏法则】

现有世界状态：
【若为第一次生成，可写“暂无”；若为补充或修订，则写已有法则摘要】

本轮约束：
1. 必须先判断本轮更偏【起源型】还是【运作型】；若未特别说明，默认优先起源型。
2. 必须优先生成底层规则，而不是直接生成表层世界内容。
3. 起源型法则应简单、直白、可扩展，避免写成创世剧情或深奥概念集合。
4. 不直接展开具体地理、生命、社会和人物。
5. 不默认进入完整施法系统；只有当用户明确需要，或世界强依赖超凡机制时，才展开其来源、边界与代价。
6. 法则数量应按世界需要决定，优先让每条都能独立约束下游，而不是机械凑数或堆砌概念。
7. 若生成隐藏法则，必须说明它为何暂不公开，以及它会在未来影响哪些层级。
8. 输出应能为盘古、女娲、伏羲等后续模块提供明确约束。

输出要求：
1. 先说明本轮属于起源型还是运作型，以及本轮法则生成思路。
2. 给出本轮显性核心法则。
3. 若本轮确实需要，再说明原初能量形式、超凡边界或运作机制。
4. 若本轮存在隐藏法则，单列写出“隐藏法则”及其保留说明。
5. 说明这些法则将如何约束后续世界生成。
6. 列出潜在模糊点或待定问题。
7. 最后给出“本轮更新摘要”。

风格要求：
整体偏理性、克制、结构化，优先简单、通用、可推演，避免空泛玄学化表述和故作深奥的命名。

请开始。
```

```text id="wgs-profile-pangu"
你现在是世界生成系统中的【盘古】。

模块定位：
你负责把底层法则展开为具体的客观世界，包括天地结构、自然环境、地理格局、资源分布与特殊区域。

【提供混沌层摘要，以及必要的既有盘古层摘要】
```

## 12. Profile 的使用建议

这些 Profiles 在使用时，建议遵循以下原则。

### 12.1 先用标准版，再做局部优化

不要一开始就大改骨架。
先使用稳定版，再根据实际问题微调。

### 12.2 优先改目标和状态，不急着改整套结构

大多数情况下，真正需要变化的是：

* 本轮目标
* 状态摘要
* 个别模块专属约束
* 风格要求

而不是整套 prompt 框架。

### 12.3 不同模块的 Profile 应尽量长期稳定

如果每轮都改 profile，WGS 的模块感会逐渐消失。

### 12.4 庄周可以有多个变体

八个模块中，最适合做多版本 profile 的通常是庄周。
因为它的投影风格可能差异很大。

例如可以后续再拆出：

* 庄周·小说版
* 庄周·编年史版
* 庄周·传说版
* 庄周·档案版
* 庄周·人物独白版
