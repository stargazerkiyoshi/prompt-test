# SLS Prompt Profiles

## 1. 文件定位

本文件用于沉淀经过实测验证的高可用 SLS prompt 模板，供调用时直接复用。

本文件属于调用资产层，不属于 Core 层、结构层、编排层、叙述层的主规范文件。

本文件中的 prompt 可随着后续实测持续迭代；若调用原则变化，应优先回看 `90-SLS-Prompt-Assembly.md`；若核心结构变化，应优先回看 `10-SLS-Core.md`、`20-SLS-Architecture.md`、`30-SLS-Teaching-Flow.md`、`40-SLS-Narrative-Guide.md`。

本文件不重新定义主规范，也不替代 `90-SLS-Prompt-Assembly.md` 的调用组织职责。

后续若需判断应优先扩展哪些 profiles，可参考 `93-SLS-Usage-Scenario-Matrix.md` 的场景分类与优先级建议。

---

## 2. 使用说明

使用这些 prompt profiles 时，建议遵循以下顺序：

1. 先按任务类型选择 profile，如完整主题学习、单概念讲解、复习总结。
2. 再按展开程度选择常规版、展开版、轻量版或压缩版。
3. 若模型默认倾向偏提纲化或偏压缩，再追加模型适配模板或调节短句。
4. 不必一次性混用全部 profile；通常选择一个主模板，再补 1 到 3 条调节短句即可。
5. 若主题、受众或输出粒度变化，优先改任务描述与输出要求，不优先重写主模板。

使用原则：

- profile 用于直接复制调用，不用于替代文件组合判断。
- profile 中出现的“展开讲解模式”等表述，视为调用增强项，不上升为 Core 原则。
- 若同一任务已经有稳定主模板，后续只做小步替换，不建议每次重新拼装整套 prompt。

## 3. 完整主题学习类

### 3.1 常规版

适用场景：

- 需要系统学习一个主题。
- 需要章节递进，但不要求特别厚的展开密度。

模板正文：

```text
请使用 SLS 的完整主题教学模式，围绕“{topic}”为我组织一次系统学习。

受众层级：{audience}
输出粒度：{chapter|lesson}

要求：
1. 先给出整体学习路径和章节递进关系。
2. 再从当前章节开始正式讲解，不要只停留在目录概览。
3. 讲解时先说明当前在解决什么问题，再建立概念与关系框架，再进入必要形式化。
4. 在章节结束时做小结，说明本章得到什么，以及下一章为什么必要。
5. 保持自然转承，不写成 checklist 式堆点。
```

使用说明或注意事项：

- 适合作为默认起点。
- 若模型默认过短，可叠加“展开讲解模式”短句。

### 3.2 展开版

适用场景：

- 用户明确要求“带我学”“详细版”“按章节展开”。
- 模型历史表现为结构正确但讲解偏薄。

模板正文：

```text
请使用 SLS 的完整主题教学模式，并启用展开讲解模式，系统讲解“{topic}”。

受众层级：{audience}
输出粒度：{chapter|lesson}

要求：
1. 先给出整体学习路径和章节递进关系，再从当前章节开始正式讲解。
2. 不要只给简洁版框架，要真正展开讲清楚，像老师在带我学。
3. 每章先解决问题，再建立概念关系，再进入必要形式化。
4. 直觉层、概念建立过程、形式化出现的动机、章节局部闭环与下一章承接都要明确展开。
5. 宁可略厚一点，也不要压缩成结构骨架。
6. 保持自然叙述和过渡，不写成提纲式概览或 checklist 式压缩输出。
7. 至少选择一个核心例子，在本章内展开走完。
```

使用说明或注意事项：

- 这是完整主题学习的高强度默认模板。
- 若输出过长，可保留展开要求，但把输出粒度收紧到单章或单节。

### 3.3 轻量版

适用场景：

- 需要快速建立主题骨架。
- 希望先判断是否继续深入学习。

模板正文：

```text
请使用 SLS 的完整主题教学模式，为“{topic}”提供一个轻量但成体系的学习入口。

受众层级：{audience}
输出粒度：{lesson}

要求：
1. 先给出主题结构和学习路径。
2. 选当前最关键的一节做正式讲解，其余部分只做必要定位。
3. 保留问题入口、概念关系、必要形式化和章节小结。
4. 以清楚为主，不必展开成完整长篇。
```

使用说明或注意事项：

- 轻量版是“保留结构意图的压缩版”，不是纯提纲版。
- 若后续继续学习，可直接切换到常规版或展开版。

## 4. 单概念讲解类

### 4.1 常规版

适用场景：

- 需要把一个概念讲明白。
- 主题范围较窄，不需要完整章节体系。

模板正文：

```text
请使用 SLS 的单概念讲解模式，讲清楚“{concept}”。

受众层级：{audience}

要求：
1. 先说明这个概念在解决什么问题，为什么需要它。
2. 再一步步建立概念定义与相关关系。
3. 在必要处给出形式化表达，并解释它在澄清什么。
4. 至少给一个例子，说明这个概念如何被使用。
5. 结尾做一个简短理解检查和一句重直觉化总结。
```

使用说明或注意事项：

- 适合绝大多数日常问答。
- 若相近概念容易混淆，可改用强辨析版。

### 4.2 增强版

适用场景：

- 用户明确要求“真正带学”。
- 模型容易退化为概览式、百科式回答。

模板正文：

```text
请使用 SLS 的单概念讲解模式，并启用展开讲解方式，带我学懂“{concept}”。

受众层级：{audience}

要求：
1. 不要只给概览，要真正把这个概念讲透。
2. 先从直觉层进入，说明它在解决什么问题，再逐步建立概念关系。
3. 概念之间的关系要一步步搭起来，不要只列定义。
4. 若需要形式化，请解释为什么现在必须引入它，它消除了什么歧义。
5. 至少选择一个核心例子，从问题进入、概念建立到结果解释完整走完。
6. 保持自然转承，不要写成提纲式摘要。
7. 结尾做一个小检查：指出常见误解，并用一句话压缩回核心直觉。
```

使用说明或注意事项：

- 这是当前单概念场景的优先推荐模板。
- 若主题有明显易混对象，可再叠加“强辨析版”的辨析要求。

### 4.3 强辨析版

适用场景：

- 相近概念经常混淆。
- 用户重点需求是“搞清差异”而不是“只听定义”。

模板正文：

```text
请使用 SLS 的单概念讲解模式，重点辨析“{concept}”与“{neighbor_concepts}”。

受众层级：{audience}

要求：
1. 先说明这些概念分别在回答什么问题。
2. 不要只并列下定义，要把它们的边界、联系和差异一步步讲出来。
3. 在相近处做对比，在关键分叉处解释为什么不能混用。
4. 若有形式化表达，请用它澄清边界，而不是单独堆公式。
5. 至少给一个对比例子，展示同一情境下这些概念各自如何使用。
6. 结尾总结：什么情况下该用哪个概念，什么情况下不能替代。
```

使用说明或注意事项：

- 适合“定义都看过，但总分不清”的场景。
- 若回答偏短，可追加“不要只给概览”“至少把一个对比例子走完”。

## 5. 复习 / 总结类

### 5.1 常规版

适用场景：

- 学完一轮后做回顾。
- 需要把零散理解重新组织成结构。

模板正文：

```text
请使用 SLS 的复习总结模式，帮我系统回顾“{topic}”。

受众层级：{audience}

要求：
1. 先概括主题结构，再回顾关键概念和它们的关系。
2. 不要只列关键词，要说明这些部分如何连起来。
3. 对必要的形式化内容，只保留最关键的澄清作用。
4. 结尾给出一份理解检查：我应该能回答哪些问题，才算这轮复习完成。
```

使用说明或注意事项：

- 适合一轮学习后的标准复盘。
- 若只需要短时记忆刷新，可改用压缩回顾版。

### 5.2 压缩回顾版

适用场景：

- 需要快速唤回记忆。
- 用户时间有限，但不希望完全退化为词条列表。

模板正文：

```text
请使用 SLS 的复习总结模式，为“{topic}”做一次压缩回顾。

受众层级：{audience}

要求：
1. 用尽量短的篇幅回顾主题主线。
2. 只保留最关键的概念、关系和一个核心例子。
3. 每一部分都要说明“它在整体里负责什么”，不要只堆关键词。
4. 结尾用三到五句话压缩成可快速复述的版本。
```

使用说明或注意事项：

- 压缩回顾版追求“可复述”，不是“全覆盖”。
- 若模型过度压缩，可追加“不要压缩成结构骨架”。

### 5.3 结构回顾版

适用场景：

- 用户最关心依赖关系、层级结构和推导路径。
- 需要反向检查“我是否真的理解了整体结构”。

模板正文：

```text
请使用 SLS 的复习总结模式，按结构回顾“{topic}”。

受众层级：{audience}

要求：
1. 先给出主题的高层结构图景，再往下回到关键定义和依赖关系。
2. 强调哪些结论依赖哪些前提，哪些概念是骨架，哪些概念是支撑。
3. 不要把复习写成散点摘要，要让我看到结构如何重构出来。
4. 在结尾用“从高层结论反推到底层定义”的方式做一次闭环。
```

使用说明或注意事项：

- 适合考试复盘、知识图谱整理、二轮学习前的结构校准。
- 若需要更自然可读，可补一条“保持自然转承”。

## 6. 模型适配类

### 6.1 偏任务执行模型适配版

适用场景：

- 模型默认更像执行任务而不是展开讲课。
- 输出常见问题是提纲化、压缩化、结构有余而讲解不足。

模板正文：

```text
请按 SLS 要求完成本次讲解，并显式启用展开讲解模式。

额外要求：
1. 不要把回答写成任务概览或提纲摘要。
2. 宁可略厚一点，也不要只剩结构骨架。
3. 先给学习路径，再进入当前部分的正式讲解。
4. 要像老师带学，而不是只列出该讲哪些点。
5. 直觉层、概念建立、形式化动机和局部闭环都要展开写出来。
6. 若需要压缩，请压缩枝节，不要压缩主线解释。
```

使用说明或注意事项：

- 这是对“结构完整但展开不足”最直接的补偿模板。
- 通常作为附加块接在主任务模板后面使用。

### 6.2 通用讲解模型适配版

适用场景：

- 模型本身较擅长自然叙述，但偶尔会弱化结构显式性。
- 需要在自然讲解和结构清晰之间加一层约束。

模板正文：

```text
请按 SLS 要求讲解本主题，并在保持自然叙述的同时，显式保证结构清楚。

额外要求：
1. 先说明当前问题，再给定义，不要直接跳到结论。
2. 讲解中持续指出当前内容与上一部分、下一部分的关系。
3. 形式化出现时，要说明它在澄清什么边界。
4. 结尾做短小收束，说明这一段得到了什么。
5. 不要因为叙述自然就省略依赖关系和结构标记。
```

使用说明或注意事项：

- 这是对“讲得顺，但结构不够显”的校正模板。
- 若用户要求更厚，可以叠加展开版主模板。

## 7. 调用调节短句

以下短句可作为可插拔增强项，按需要追加到主 prompt 后：

- 默认采用展开讲解模式。
- 不要只给概览，请进入正式讲解。
- 宁可略厚一点，也不要压缩成结构骨架。
- 不要写成提纲式概览或 checklist 式摘要。
- 请先给学习路径，再进入当前部分。
- 请至少选一个核心例子展开走完。
- 不要只列概念，要把关系一步步建立出来。
- 形式化出现时，请解释它的动机和作用。
- 在相近概念之间做适度辨析。
- 每章形成局部闭环，并说明下一步为什么必要。
- 每段结束时，用一句话收束当前得到的结论。
- 若需要压缩，请压缩枝节，不要压缩主线解释。
- 请把新结论回扣到开头的问题。
- 请保留自然转承，不要只堆定义和公式。
- 请指出一个常见误解，并顺手纠正它。

## 8. 维护建议

后续若新增 profile，建议保持以下记录口径：

1. 先标注适用任务类型。
2. 再标注它解决的默认输出倾向问题。
3. 仅沉淀经过实测后可重复复用的模板。
4. 不把临时试验句、零散备注直接写入主 profile 区。

本文件的目标是稳定复用，而不是收纳所有 prompt 草稿。
