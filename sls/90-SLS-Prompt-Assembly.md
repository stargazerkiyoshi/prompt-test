# SLS Prompt Assembly（第二轮收束版）

## 1. 调用层定位

本文件只定义调用组合方式与组装流程。

调用层不重复定义 Core 层、结构层、编排层、叙述层的本体规范。

具体可直接复用的 prompt 模板见 `92-SLS-Prompt-Profiles.md`；本文件不承载 prompt 资产正文。

---

## 2. 组合原则

1. 先判定任务类型，再选择文件组合。
2. 以最小必要组合满足任务，避免全量拼接。
3. 组合时只引用他层规范，不在调用层改写他层原则。
4. 输出前做越层检查，确保职责单一。
5. 涉及术语时引用 `25-SLS-Terms.md`，避免术语漂移。

---

## 3. 任务模式与文件组合

### 3.1 完整主题教学

推荐组合：Core 层 + 结构层 + 编排层 + 叙述层。

### 3.2 单概念讲解

推荐组合：Core 层 + 结构层 + 叙述层。

### 3.3 复习与总结

推荐组合：Core 层 + 结构层。

可选追加：编排层（按路径回顾）或叙述层（提升可读性）。

### 3.4 简化讲解

推荐组合：Core 层 + 叙述层。

### 3.5 知识生成

推荐组合：Core 层 + 结构层 + 编排层。

### 3.6 知识工程场景

推荐组合：Core 层 + 结构层 + 调用层。

说明：知识工程层暂不拆分独立文件，按调用模式最小化承载。

---

## 4. 组装步骤

1. 明确任务目标、主题范围、受众层级。
2. 选择对应任务模式与文件组合。
3. 指定输出粒度（章节级、节级、问答级）。
4. 写入约束引用（例如：Core 层原则编号、编排层规则编号）。
5. 生成后检查是否越层与是否缺层。

---

## 5. 模型适配与展开讲解模式

### 5.1 经验定位

最近实测表明，不同模型即使接收相同的 SLS 调用约束，默认输出倾向仍会不同。

调用层需要承认这种差异，并通过任务模式、输出要求与展开指令做适配；不应把模型默认风格差异直接上升为 Core 层或结构层修改理由。

### 5.2 常见输出倾向

- 偏叙述生成的模型，通常更容易给出自然过渡、直觉铺垫与展开说明，但偶尔会弱化结构显式性。
- 偏任务执行的模型，通常更容易满足结构正确、依赖清楚、章节齐整，但默认输出更压缩，容易退化为“结构完整但展开不足”。
- 同一模型在不同上下文密度下，也可能在“自然展开”与“压缩执行”之间摆动。

因此，若问题首先表现为“讲得太短、结构对但不够带学”，优先判断为调用适配问题，而不是立刻修改主规范。

### 5.3 对偏任务执行模型的调用建议

当模型更偏任务执行、默认回答偏压缩时，调用层建议显式补足以下约束：

1. 明确任务不是“提纲概览”，而是“展开讲解”。
2. 明确要求先给学习路径，再进入当前章节的正式讲解。
3. 明确要求直觉、概念建立过程、形式化动机、局部闭环都要展开。
4. 明确要求避免 checklist 式压缩输出，保持自然转承。
5. 如是完整主题教学，优先使用 Core 层 + 结构层 + 编排层 + 叙述层的完整组合，不做过早裁剪。

必要时可在输出要求中直接写明：

- “不要只给简洁版框架，要真正展开讲清楚。”
- “像老师带学，而不是只给提纲式概览。”
- “每章先解决问题，再建立概念关系，再进入必要形式化。”

### 5.4 展开讲解模式

“展开讲解模式”用于处理以下情况：

- 用户明确要求系统学习、按章节推进。
- 用户明确要求详细版、带学版、展开版。
- 模型历史表现显示结构正确但默认压缩。

建议将其视为调用层的标准增强项，而不是新的 Core 原则。

标准调用语句可写为：

```text
请使用 SLS 的完整主题教学模式，并启用展开讲解模式。
先给出整体学习路径和章节递进关系，再从当前章节开始正式讲解。
不要只给简洁版框架，而要真正展开讲清楚，像老师在带我学。
直觉层、概念建立过程、形式化出现的动机、章节局部闭环与下一章承接都要明确展开。
保持自然叙述和过渡，不写成 checklist 式压缩提纲。
```

若需要更强约束，可再追加：

```text
若模型默认倾向压缩，请优先增加解释展开度，而不是压缩为结构提纲。
```

---

## 6. 调用注意事项

- 不把各层原文整段粘贴为单次 prompt。
- 不在调用层新增“伪原则”。
- 不用调用技巧替代结构约束。
- 任务变化时优先改组合，不优先改 Core。
- 模型适配优先在调用层处理，不把默认输出风格误判为主规范缺陷。
- 调用验证前先对照 `91-SLS-Validation-Scenarios.md` 选择场景与观察点。
