# SLS Prompt Assembly（第二轮收束版）

## 1. 调用层定位

本文件只定义调用组合方式与组装流程。

调用层不重复定义 Core 层、结构层、编排层、叙述层的本体规范。

---

## 2. 组合原则

1. 先判定任务类型，再选择文件组合。
2. 以最小必要组合满足任务，避免全量拼接。
3. 组合时只引用他层规范，不在调用层改写他层原则。
4. 输出前做越层检查，确保职责单一。
5. 涉及术语时引用 `25-SLS-Terms.md`，避免术语漂移。

---

## 3. 任务模式与文件组合

### 3.1 完整主题教学

推荐组合：Core 层 + 结构层 + 编排层 + 叙述层。

### 3.2 单概念讲解

推荐组合：Core 层 + 结构层 + 叙述层。

### 3.3 复习与总结

推荐组合：Core 层 + 结构层。

可选追加：编排层（按路径回顾）或叙述层（提升可读性）。

### 3.4 简化讲解

推荐组合：Core 层 + 叙述层。

### 3.5 知识生成

推荐组合：Core 层 + 结构层 + 编排层。

### 3.6 知识工程场景

推荐组合：Core 层 + 结构层 + 调用层。

说明：知识工程层暂不拆分独立文件，按调用模式最小化承载。

---

## 4. 组装步骤

1. 明确任务目标、主题范围、受众层级。
2. 选择对应任务模式与文件组合。
3. 指定输出粒度（章节级、节级、问答级）。
4. 写入约束引用（例如：Core 层原则编号、编排层规则编号）。
5. 生成后检查是否越层与是否缺层。

---

## 5. 调用注意事项

- 不把各层原文整段粘贴为单次 prompt。
- 不在调用层新增“伪原则”。
- 不用调用技巧替代结构约束。
- 任务变化时优先改组合，不优先改 Core。
- 调用验证前先对照 `91-SLS-Validation-Scenarios.md` 选择场景与观察点。

---

## 6. 调用模板（草稿）

```text
任务模式：{mode}
目标主题：{topic}
受众层级：{audience}
输出粒度：{chapter|lesson|qa}
文件组合：{core, architecture, flow, narrative}
约束引用：{core_rules, flow_rules, narrative_rules}
输出要求：{deliverables}
```
