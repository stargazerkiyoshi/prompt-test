# SLS 分层索引（第二轮收束版）

## 1. 文件定位

本文件只用于导航与维护说明，不承载核心原则、结构定义、编排规则、叙述规则或调用细节。

---

## 2. 分层说明

- `10-SLS-Core.md`（Core 层）
  目标、原则、边界、适用范围、层级区分。

- `20-SLS-Architecture.md`（结构层）
  教学单元的认知部件、部件作用、部件关系、结构完备性。

- `25-SLS-Terms.md`（术语支持文件）
  术语定义、易混辨析、命名约定。

- `30-SLS-Teaching-Flow.md`（编排层）
  默认推进路径、阶段任务、交错规则、执行节奏、编排错误。

- `40-SLS-Narrative-Guide.md`（叙述层）
  叙述原则、表达倾向、转承收束、自然讲解要求。

- `90-SLS-Prompt-Assembly.md`（调用层）
  文件组合方式、任务模式、组装步骤、调用注意事项。

- `91-SLS-Validation-Scenarios.md`（验证准备）
  调用验证场景设计与观察点。

- `99-SLS-Changelog.md`
  规范演化与关键决策记录。

- `CHANGELOG.md`（历史兼容）
  早期变更记录文件。

## 4. 维护规则

1. 不同层的内容只写在对应文件，避免双重归属。
2. Core 层保持原则稳定，执行细节下沉到编排层或叙述层。
3. 结构层定义“由什么构成”，编排层定义“如何展开”。
4. 调用层只做组合，不反向改写其他层规范。

---

## 5. 术语约定

- “层”：文件级规范层级（Core 层、结构层、编排层、叙述层、调用层）。
- “部件”：教学单元内部组成（如直觉图像部件、形式化部件）。
- “编排”：教学推进与展开顺序。
- “叙述”：表达方式与讲解呈现。
- “形式化”：精确表达与可推导化表达。
