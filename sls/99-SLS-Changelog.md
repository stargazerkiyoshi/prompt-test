# SLS Changelog

## 1. 文件定位

本文件记录 SLS 的结构演化与关键决策，用于长期维护与版本审计。

本文件不是用户指南，也不承载教学规则本体。

---

## 2. 当前版本状态

- 当前规范基线：`v1.4`。
- 当前形态：分层规范体系（Core / Architecture / Teaching Flow / Narrative Guide / Prompt Assembly）。
- 当前状态：已完成第二轮收束，进入配套定稿与调用验证准备阶段。

---

## 3. 重构摘要（单文件到分层）

旧版 SLS 为单文件混合规范，原则、结构、流程、叙述、调用内容耦合较高。

重构后，按职责拆分为多文件分层体系：

- `10-SLS-Core.md`
- `20-SLS-Architecture.md`
- `30-SLS-Teaching-Flow.md`
- `40-SLS-Narrative-Guide.md`
- `90-SLS-Prompt-Assembly.md`

并增加索引与配套文件以支持维护与验证。

---

## 4. 第一轮记录：拆分与迁移

第一轮重点：

- 将旧版单文件内容迁移到分层文件。
- 建立基础职责边界。
- 标注重复、层级不清与待定项，作为后续收束输入。

结果：完成初步可用分层，但存在跨层重复与术语混用。

---

## 5. 第二轮记录：去重、定责、术语统一

第二轮重点：

- 清理重复内容与层级不清段落。
- 固定“Core 原则 + 编排层执行”的双层收束关系（节奏控制、结构重构一致）。
- 将“七模块/七部件”收束为“标准完整教学单元的参考结构”。
- 统一主术语为：层、部件、编排、叙述、形式化。
- 将版本说明与 Core 主体分离。

结果：主体系职责稳定，维护成本显著下降。

---

## 6. 当前保留决策与后续方向

当前保留决策：

- 知识工程层暂不拆分为独立规范文件。
- “模块”退为历史兼容术语，不作为主术语。

后续方向：

- 按 `91-SLS-Validation-Scenarios.md` 执行调用验证。
- 依据验证结果做小步迭代，不回退到混合单文件形态。
