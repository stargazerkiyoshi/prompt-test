# 第 6 章 第 3 课：Prompt 与生成，本质上是在做条件续写

上一课已经解释了为什么下一词预测会学到很多东西。现在第 6 章的最后一个关键问题来了：

模型在真正使用时，到底是怎样从“预训练里学到的东西”走到“用户眼前的回答”的？

这里最容易被神秘化的两个词，就是 prompt 和 generation。尤其是 prompt，很多人会不自觉把它想成某种神奇咒语，好像一句话写得巧，模型就突然“被激活”了。其实如果把前面几章的主线回收一下，就会发现这件事并不神秘。

所以这一课要解决的问题是：

prompt 到底在给模型什么，生成过程又为什么是一步步展开的？

## 先把 prompt 放回前面已经建立好的逻辑里

从第 6 章前两课走到这里，你已经知道两件事。

第一，Decoder-only 模型是在前缀条件下逐步生成的。

第二，下一词预测让模型学到了大量“在某种前缀下，后面文本通常如何继续”的模式。

那 prompt 到底是什么？

其实非常朴素：

prompt 就是你在推理时主动构造给模型的前缀条件。

它不是在模型外面附魔，而是在模型里面触发“接下来应该延续哪类文本模式”的条件输入。

如果你问一个问题，prompt 就在告诉模型：当前最像“问题之后接答案”的 continuation 场景。

如果你写“把下面句子翻译成英文：……”，prompt 就在告诉模型：当前最像“翻译任务模板”的 continuation 场景。

如果你给几组输入输出样例，再接一个新输入，prompt 就在告诉模型：当前最像“照前面样例模式继续完成一个新案例”的 continuation 场景。

所以 prompt 的本质不是魔法控制，而是：

用一个前缀，把模型在预训练中学到的某类延续模式调出来。

## 为什么 prompt 会对结果影响这么大

一旦把 prompt 理解成“前缀条件”，它为什么重要就很好解释了。

因为 Decoder-only 模型的整个工作方式，本来就是根据前缀来决定下一个 token 的概率分布。那前缀一变，后面每一步的分布都会变。于是整个回答风格、信息组织、任务边界，都会跟着变。

也就是说，prompt 不是在最后一层轻轻推一下结果，而是在从第一步起就改变了模型的条件空间。

举个最简单的例子。

如果你只输入：

“Transformer 是什么？”

模型会进入一种开放问答 continuation 模式。

如果你输入：

“请用高中生能懂的方式，分三段解释 Transformer 是什么。”

模型进入的就不只是问答模式，还多了受众层级、表达风格和输出结构约束。

如果你再输入示例：

“问题：什么是微积分？
回答：……
问题：什么是 Transformer？
回答：”

模型就会更强地进入“仿照前例给出说明”的 continuation 模式。

所以 prompt 影响大，不是因为它神秘，而是因为它直接决定了模型从哪类前缀分布开始继续生成。

## 为什么生成一定是一词一词往后写

现在再来看 generation 本身。

很多人第一次使用大语言模型时，虽然知道它是逐词生成，但心里还是会忍不住想：

模型是不是其实脑中已经有了完整答案，只是一个词一个词往外吐？

更准确的理解应该是：模型在每一步上都只显式决定当前下一个 token 的概率分布，然后选出或采样出一个 token，把它接到上下文后面，再进入下一步。

所以生成不是“先想好全文，再逐字打印”，而是：

每走一步，都在更新新的前缀；新的前缀又决定下一步的分布。

这和前面讲的自回归结构完全一致。当前输出一旦被写出来，它立刻就会成为后续条件的一部分。于是生成过程天然是一条滚动展开的链。

这也解释了为什么模型有时会“越写越偏”。因为前面生成出的 token 一旦稍微偏离，它会反过来成为后面步骤的新条件，误差就可能被不断继承和放大。

## 采样到底在干什么：不是随机乱写，而是在分布上做选择

接下来要讲的，是很多人经常直觉混淆的地方。

模型每一步其实不是只给一个词，而是给一个对所有可能 token 的概率分布。然后系统需要决定：到底怎样从这个分布里选出当前要写的那个 token。

最直接的办法是总选概率最高的那个，这通常叫 greedy。这样做的好处是稳定、确定，但也可能让输出过于保守、重复，或者早早走进一个局部模式。

另一类办法是按分布采样。也就是说，不总是只拿第一名，而是让高概率 token 更容易被选中，但也给其他较合理 token 一定机会。

这时候生成看起来就更灵活，但也更容易发散。

所以采样不是“随机乱来”，而是：

在模型已经给出的概率分布之上，决定输出的探索程度。

这一步很重要，因为它说明“模型能力”和“解码策略”并不是一回事。模型负责给分布，解码策略负责从分布里挑词。

## temperature 在调什么

现在就能顺势讲 temperature 了。

temperature 的作用，本质上是在改变概率分布的尖锐程度。

如果 temperature 更低，分布会更尖，高概率 token 更占优势，输出更稳定、更保守。

如果 temperature 更高，分布会更平，更多候选 token 获得机会，输出更发散，也更有多样性。

请注意，temperature 不是在给模型加知识，也不是在改变模型已经学到的结构。它只是在调：

模型当前概率分布被用多“硬”还是多“软”地执行。

所以当你看到“同一个 prompt，temperature 不同，回答风格也不同”，不要把它理解成模型本体突然变了，而要理解成：

同一个分布，被用不同探索强度去解码了。

## 为什么同一个 prompt 有时稳定、有时飘

这一点也很值得讲清楚，因为它和实际使用体验直接相关。

同一个 prompt 有时回答很稳，有时回答会发散，原因通常不只一个。

一部分来自 prompt 本身是否把任务边界、语气和结构条件给清楚了。前缀越清楚，模型进入的 continuation 模式越集中。

一部分来自模型内部在这个问题上的概率分布本来就可能不尖锐。如果多个后续方向都看起来还行，生成自然更容易不稳定。

再一部分来自解码策略。采样、temperature、top-k、top-p 等设置都会影响最终落下来的 token。

所以所谓“飘”，往往不是一句“模型不行”就能解释，而是：

前缀条件不够收紧、模型分布本身不够确定、解码策略又给了更大探索空间，这几件事叠在一起。

## 用贯穿例子把本课走完

还是用那句：

“这只动物没有过马路，因为它太累了。”

如果你把这句话一部分放进 prompt，让模型继续写，比如：

“这只动物没有过马路，因为它太”

模型接下来会给出很多 token 的概率分布。“累”很可能较高，因为它和前缀中的因果解释、代词结构、常见表达模式都比较一致。

此时如果用 greedy，模型大概率直接选“累”。

如果用较高 temperature 采样，别的少数合理 token 也可能被抽中，后续发展可能更灵活，也可能更不稳定。

这个例子看似很小，却已经把 prompt 和 generation 的本质都压出来了：

prompt 提供前缀条件，生成在这个条件下逐步展开，而每一步展开都来自当前概率分布上的选择。

## 一点必要的形式化轮廓

现在可以给出最小形式化总结。

给定 prompt，也就是前缀

$x_1, x_2, \dots, x_t$

模型先计算下一个 token 的条件分布：

$P(x_{t+1} \mid x_1, \dots, x_t)$

然后根据某种解码策略选出一个具体 token，接到前缀后面，形成新的前缀：

$x_1, x_2, \dots, x_t, x_{t+1}$

接着再继续下一步。

你会发现，这其实就是预训练目标在推理时的直接延续。训练时模型学的是“给定前缀预测下一个 token”，推理时模型做的也是“给定前缀生成下一个 token”，只是这时候前缀里有一部分是用户 prompt，另一部分是模型自己已经生成出的内容。

## 本课的局部闭环

我们现在把这节课收一下。

prompt 的本质，是人为构造模型的前缀条件；generation 的本质，是在这个前缀条件下逐步做下一个 token 的分布预测与选择。采样和 temperature 不是神秘附加物，而是在控制模型给出的分布怎样被执行。于是，大语言模型的使用过程本质上就是：用 prompt 设定 continuation 场景，再让模型沿这个场景一步步写下去。

如果把本课压成一句最关键的话，那就是：

prompt 负责定条件，generation 负责在这个条件下滚动展开，而每一步都只是下一 token 分布上的一次选择。
