# 第 6 章 第 2 课：为什么下一词预测会学到远超“续写”的东西

上一课已经解释了为什么 Decoder-only 架构能够单独成立。现在真正更核心的问题来了：

为什么一个看起来这么朴素的训练目标，竟然能支撑出如此强的模型能力？

表面上看，下一词预测只是让模型不断回答：

“给定前面的 token，后面最可能是什么？”

这听上去好像只是一个续写游戏，甚至容易给人一种“这不就是高级自动补全吗”的印象。但如果你真的顺着这个任务往深处看，就会发现它逼模型学到的东西远不止局部接词。

所以这一课要解决的问题是：

下一词预测为什么不是一个低级任务，而会迫使模型学习大量语言结构、事实模式和任务形式？

## 先从一个最直接的事实开始：想预测下一个词，往往必须先理解前面发生了什么

还是看句子：

“这只动物没有过马路，因为它太……”

如果现在让模型预测下一个词，它当然可能猜“累”。但这个猜测不是凭空来的。为了让“累”显得高概率，模型至少需要掌握几层东西。

它要知道“因为”后面通常会跟原因解释，而不是任意无关成分。

它要知道“它”在这里不是悬空代词，而是在回指前文中的某个对象。

它要知道“没有过马路”和后面的原因解释在语义上是连得起来的。

它还要知道在语言分布中，“因为它太累了”是一种比许多随机续写更合理的延续。

也就是说，想把下一个 token 预测对，模型往往不能只盯着前面几个字，而必须先形成对前缀整体结构的某种理解。

这就是下一词预测最容易被低估的地方：

目标形式虽然局部，但为了把这个局部目标做好，模型必须学习大量全局结构。

## 为什么语法、语义和世界模式都会被卷进来

我们可以把这个问题再拆细一点。

如果你想稳定预测后一个词，首先会被迫学习语法模式。因为很多 token 的高概率与否，直接取决于句法结构。比如当前位置更适合接名词还是动词，更适合接标点还是从句，都和语法组织高度相关。

其次，你还会被迫学习语义连贯。因为只有语法还不够，很多句子语法上能接，但语义上非常怪。模型如果不学会语义搭配和事件合理性，就很难在真实文本分布里把概率压到正确方向。

再往上，你甚至会被迫吸收大量世界模式。为什么？因为真实文本里很多下一词选择，离不开常识、领域知识、社会习惯、文档结构和任务模板。比如问题后面常跟答案，定义后面常跟解释，新闻里时间地点有常见搭配，对话里上一轮内容会影响下一轮回答。

于是，下一词预测虽然看起来只在训练一个局部输出，但它像一个吸力很强的目标，把语言中的各层规律都卷了进来。

所以更准确的说法不是：

“模型学会了预测词。”

而是：

“模型为了做好词预测，被迫学会了支撑这种预测的大量结构和模式。”

## 这和“背答案”有什么根本区别

到这里很多人会问：那这是不是还是一种大规模记忆？模型是不是只是见过很多句子，所以把后面背出来了？

这个问题不能简单回避。

模型当然会吸收大量训练文本中的统计规律，也可能记住一些高频模式甚至具体片段。但如果你把它理解成纯背诵，就解释不了很多现象。因为真实训练目标不是让模型死记一条固定后续，而是在海量不同上下文中不断学习：

什么样的前缀，会怎样改变后续 token 分布。

这里的关键不是单条句子，而是分布模式。模型在无数样本中看到：

- 哪些语法结构通常如何收尾；
- 哪些问题形式通常如何回答；
- 哪些事件描述通常如何延续；
- 哪些文本体裁有怎样的展开方式。

于是它学到的不是单一答案，而是一套高维条件分布上的组织规则。

这就是为什么下一词预测最后会长出某种“会做任务”的感觉。因为许多任务本身就能被写成“在某种上下文条件下，下一段文本该怎么继续”的问题。

## 为什么大规模数据和参数会把这种效应放大

下一词预测并不是魔法。它之所以在今天表现得如此强，很大程度上还依赖两个放大条件：

数据量足够大，模型容量也足够大。

数据量大，意味着模型会看到极其丰富的语言现象、任务模板、知识分布和文档结构。

参数量大，意味着模型有能力把这些复杂规律压进自己的表示系统里，而不只是学到很浅的局部搭配。

于是，随着规模增加，模型不再只是在学“短距离搭配”，而越来越像是在学：

如何利用长上下文；
如何组织多步结构；
如何在不同文体、不同任务模板里继续生成合理文本。

这就是为什么“下一词预测”这个目标本身没变，但大模型和小模型表现会有巨大差异。不是目标突然变魔法了，而是规模让目标能逼出更高层次的规律吸收。

## 用任务迁移看这一点为什么尤其明显

还有一个特别能说明问题的现象是任务迁移。

比如你不给模型专门训练一个“问答模块”，而只是给它很多文本，其中包含问答、解释、定义、总结、翻译等各种形式。模型在做下一词预测时，会不断看到这样的模式：

问题之后常接答案；
说明之后常接例子；
命令之后常接执行结果；
“把下面句子翻译成英文：”之后常接英文句子。

久而久之，模型就会把这些模式学成一种“任务条件续写结构”。于是当用户在 prompt 里给出类似格式时，模型会顺着这种已学到的文本结构继续生成。

这就是为什么很多任务能力最后能被统一吸收到“给定前缀继续写下去”这个接口里。

从这个角度看，prompt 不是什么外层黑科技，而更像是：

人为构造一个前缀条件，让模型把自己在预训练中学到的条件续写模式调出来。

这一步会在下一课正式展开。

## 用贯穿例子把本课走完

还是看这句：

“这只动物没有过马路，因为它太……”

如果模型最后要把“累”赋予较高概率，它至少得吸收下面这些规律中的若干部分：

- “因为”后面更可能接解释成分；
- “它”必须和前文某个对象形成稳定关系；
- 整句描述的是一个未发生事件和一个原因解释；
- “太累了”在语义上是合理延续，而很多别的词不是；
- 在真实文本分布里，这种表达模式本来就常见。

你会发现，这里面已经不只是局部续写，而是句法、语义、指代、常识模式同时进入。

所以如果下一词预测能在海量文本上不断重复这类练习，它最后学到的当然不只是一串局部拼接技巧。

## 一点必要的形式化轮廓

现在可以给出最小形式化总结。

下一词预测目标通常写成最大化整段文本的联合概率：

$P(x_1, x_2, \dots, x_T) = \prod_{t=1}^{T} P(x_t \mid x_1, \dots, x_{t-1})$

这个公式看上去只是把整段文本拆成一步步条件概率，但它背后的学习压力非常大。因为要把这个乘积里的每一项都做好，模型就得学会：

如何利用前缀中的结构、知识和模式，把后一个 token 的分布尽量压到正确方向。

所以公式虽然短，学习要求却非常厚。

## 本课的局部闭环

我们现在把这节课收一下。

下一词预测之所以强，不是因为“预测一个 token”本身多高级，而是因为为了把这个局部目标做好，模型被迫吸收语法、语义、知识、文体和任务模板等大量规律。随着数据规模和模型容量增大，这种吸收会越来越深，于是模型表现出来的能力也远远超出“接着写”表面的直观印象。

如果把本课压成一句最关键的话，那就是：

下一词预测的表面目标很局部，但它逼模型学习的支撑结构非常全局。
