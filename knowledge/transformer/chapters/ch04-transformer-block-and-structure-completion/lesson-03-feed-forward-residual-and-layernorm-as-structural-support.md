# 第 4 章 第 3 课：前馈、残差与归一化是在给整层“立结构”

到这一课为止，第 4 章已经补了两个关键缺口。位置编码给了 self-attention 顺序感，多头注意力给了它并行的关系视角。现在还剩下一个很自然的问题：

如果一个位置已经能带着顺序感、从多个视角去看整句，那一个标准 Transformer block 是不是已经成立了？

还差最后一截。

因为到目前为止，attention 做的主要还是一件事：

让各个位置彼此交换信息。

这件事当然关键，但它还没有完整回答两个后续问题：

第一，信息交换完之后，当前位置自己怎样把这些信息真正加工成更有表达力的表示？

第二，如果这样的更新要一层层反复堆叠，整个 block 怎样保持可训练、可传递、可稳定？

所以这一课引入前馈网络、残差连接和层归一化，不是在平地上突然加新名词，而是在把“一个标准 block 还缺什么”这条链走完。

## 先把这一课放回正确层级

这里先把层级再摆正一次。我们现在讲的仍然不是完整 Transformer 系统，而是单个标准 block 的内部收尾。

第 3 章讲的是发动机：self-attention 怎样做关系读取。

第 4 章前两课补的是这个发动机还缺的两样东西：顺序感和多视角。

而这一课要补的是 block 级的最后两类能力：

- 一个位置在读完别人之后，怎样继续加工自己；
- 一个这样的更新单元，怎样被稳定地重复很多层。

只要这个层级感不丢，你就会发现这节课其实很顺：先补“位置内部加工”，再补“深层可堆叠性”。

## 为什么 attention 后面自然会轮到前馈网络

self-attention 很擅长决定“我现在该从谁那里吸收多少信息”。当前位置通过权重从其他位置取回内容，得到一个新的上下文化表示。

但你仔细看就会发现，这一步更像是“把相关信息找过来、混进来”，还不等于“把混进来的信息充分加工好”。

换句话说，attention 更偏向回答：

我该看谁？

而它没有完全回答：

看完之后，我在自己这个位置上该怎样进一步变形、提炼、放大和压缩这些信息？

这正是前馈网络出现的地方。

前馈网络通常逐位置独立作用。这里的“独立”不是说它回到了没有上下文的状态，而是说：

上下文相关信息已经在 attention 阶段被带进来了；
现在轮到每个位置基于自己已经拿到的上下文，做更充分的非线性加工。

所以 block 里的这两部分可以先粗略分工成：

- attention 负责跨位置交流；
- feed-forward 负责位置内部加工。

这不是重复劳动，而是非常自然的先后节奏。

## 为什么“逐位置独立”不是退步

很多人第一次看到 FFN 都会疑惑：前面 attention 好不容易把全句联系起来，为什么后面又回到每个位置各自处理？

因为 attention 最昂贵、最关键的全局步骤已经做完了。到 FFN 出场时，每个位置手里的表示已经不是孤立词向量，而是一个吸收了上下文的结果。

这时模型真正需要的，不是再来一次全局互看，而是把当前位置已经拿到的上下文，进一步变成更适合后续层使用的内部特征。

这很像你读一句话时，先通过整句关系判断“它”大概指向“动物”，再在脑中把这个位置加工成更稳定的角色理解，而不是每想一步都重新扫整句。

所以 FFN 的意义不是把模型拉回局部，而是让“已经上下文化的位置表示”真正变得更有表达力。

## 但有了 attention 和 FFN，为什么还不够

如果到这里停下，一个单层 block 的功能已经很像样了：

- attention 让位置彼此交流；
- FFN 让每个位置继续消化这些交流。

可 Transformer 不是只用一层。它真正的力量，来自很多这样的 block 可以一层层往上堆。

一旦进入“深层堆叠”这个背景，问题就变了。我们不再只问“这一层算得对不对”，还要问：

这一层的更新能不能顺利传给下一层？
原有表示会不会被层层改写得难以追踪？
数值分布会不会一路漂移，导致训练越来越不稳？

残差连接和层归一化，就是在这个时候自然出现的。

## 残差连接是在给深层更新保留主通路

残差连接的核心思想很朴素：一层不必每次都把输入彻底推翻重来，而是可以保留原输入通路，再在这个基础上学习一个增量修正。

这件事为什么重要？因为深层模型里，如果每一层都必须完全重写表示，信息和梯度都更难稳定传递。很多本来有用的结构，会在反复变换中越来越难追踪。

有了残差之后，一层的工作方式会变成：

不是“我重新发明一个表示”，
而是“我在已有表示上做一次可控更新”。

这样，深层堆叠才真正变得现实。

所以残差连接不是训练时顺手加的辅助技巧，而是在回答：

block 怎样才能被重复很多次，而不把已有表示越改越散？

## 层归一化是在给深层计算链维持稳定工作区间

光有残差还不够。因为随着 attention、FFN 和残差不断叠加，各层表示的数值分布仍然可能一路漂移。

层归一化的作用，就是让关键子层之后的表示保持在更可控的范围内。你可以把它理解成：

每做完一次重要更新，就把表示拉回一个更稳定、更容易继续计算的工作区间。

这一步在深层结构里尤其重要。因为后面的层不是看语义标签，而是直接吃前面的数值表示。如果这些表示尺度越来越飘，训练就会变得敏感甚至不稳定。

所以 layer norm 不是孤立的数学装饰，而是在维护整条深层计算链的数值生态。

也正因为如此，残差和归一化通常要放在一起理解：

- 残差负责保留稳定主通路；
- 层归一化负责维持表示分布的可控性。

## 现在把它们重新放回标准 block 里看

到这里，一个标准 Transformer block 的内部节奏就基本完整了。

先有多头 self-attention，让每个位置带着顺序感、从多个关系视角和整句其他位置交换信息。

然后通过残差与归一化，把这一步稳定并入主表示通道。

接着进入前馈网络，对每个位置已经上下文化过的表示做更深的非线性加工。

之后再次通过残差与归一化，让这一轮位置内部加工也稳定地回到主通路。

所以一个标准 block 不是“attention 后面顺手接点别的”，而是非常清楚的两阶段更新单元：

先跨位置交流；
再位置内加工；
每一步都带着稳定深层堆叠所需的结构保护。

## 用贯穿例子把这一节走完

还是那句：

“这只动物没有过马路，因为它太累了。”

假设模型正在更新“它”这个位置。经过 attention，它已经从“动物”“因为”以及事件背景里吸收了一些相关信息。到这一步，它确实不再只是一个薄薄的代词表示了。

但如果停在这里，它仍然只是“混合过信息”。模型还需要把这种混合结果进一步加工成更适合后续层使用的内部特征。也许某些维度要被放大，也许某些组合模式要被压缩得更稳定，这就是 FFN 在做的事。

而如果这种“交流之后再加工”的过程要重复很多层，没有残差通路，原有表示就容易在反复更新中变得难追踪；没有 layer norm，数值分布也容易越来越漂。

所以从这个例子里，你已经能把分工看得很清楚：

attention 负责把相关信息找过来；
FFN 负责把找来的东西在当前位置真正加工好；
residual 和 layer norm 负责让这种更新能稳定地层层积累。

## 一点必要的形式化轮廓

现在可以给出最小骨架，但重点仍然是看结构节奏，而不是背模板。

一个标准 Transformer block 常常可以先粗略写成两段：

$H_1 = LayerNorm(X + MultiHeadAttention(X))$

$H_2 = LayerNorm(H_1 + FFN(H_1))$

这里最值得记住的不是符号，而是这两段分别在干什么：

- 第一段做跨位置交流，并把结果稳定接回主通路；
- 第二段做逐位置加工，并再次稳定接回主通路。

这一节奏一旦理解，后面无论看到 Pre-LN 还是 Post-LN 变体，你都不会丢掉主线。

## 本课的局部闭环

我们现在把这一课收一下。

前两课补的是 self-attention 发动机还缺的顺序感和多视角，这一课补的是标准 block 还缺的“位置内部加工”和“深层可堆叠性”。前馈网络让每个位置真正消化上下文，残差连接让深层更新变成增量修正，层归一化让整条计算链维持稳定分布。它们合在一起，才让 self-attention 不只是一个能工作的机制，而变成一个真正可堆叠、可训练的标准结构单元。

如果把本课压成一句最关键的话，那就是：

attention 让位置彼此交流，FFN 让每个位置消化这些交流，而残差与归一化让这种更新能稳定层层累积。
