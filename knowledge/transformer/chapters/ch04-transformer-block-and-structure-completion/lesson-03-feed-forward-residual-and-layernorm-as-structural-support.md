# 第 4 章 第 3 课：前馈、残差与归一化是在给整层“立结构”

到这一课为止，第 4 章已经补了两个关键缺口。位置编码让 self-attention 终于有了顺序感，多头注意力让同一层能并行地从多种关系视角阅读序列。现在看起来，Transformer 的核心是不是已经差不多完整了？

还不够。

因为到目前为止，attention 做的核心事情仍然主要是：

让各位置彼此交换和混合信息。

这件事很关键，但它并不自动等于“每个位置都被充分加工了”。同样地，一旦网络开始加深，如果没有额外结构保护，训练过程也不会自动稳定。

所以完整 Transformer block 里，还必须有前馈网络、残差连接和层归一化。很多人第一次看到这些组件，会觉得它们像“工程配件”，比 attention 本体次要。但实际上，它们的地位很高。因为它们回答的是两个 self-attention 自己没完全解决的问题：

第一，信息混合完之后，每个位置怎样进一步被非线性加工？

第二，很多层堆起来之后，整个系统怎样保持可训练、可传递、可稳定？

这节课就要把这两件事讲清楚。

## 先解决前馈网络为什么必须出现

我们先看第一个问题。

self-attention 非常擅长做“位置之间的信息交换”。当前位置通过权重从其他位置吸收信息，得到新的上下文化表示。这很好，但你仔细想会发现，它更偏向一种“混合”和“重组”，而不是深入加工。

换句话说，attention 很像是在说：

你现在该从谁那里吸收多少内容。

但它没有充分回答另一件事：

吸收完这些内容之后，当前位置自己应该怎样进一步变形、提炼、放大某些模式、压缩某些模式？

这正是前馈网络出现的位置。

前馈网络通常是逐位置独立作用的。注意这句话很重要。它不像 attention 那样在位置之间建联系，而是对每个位置已经得到的表示，单独做一轮更深的非线性变换。

所以你可以把 block 里的这两部分先粗略分工成：

- attention 负责跨位置交流；
- feed-forward 负责位置内部加工。

这个分工非常自然。因为一句话理解不能只靠“我看了谁”，还要靠“我把看到的东西在自己这里加工成什么”。

## 为什么“逐位置独立”不是退步，反而是必要补足

有人第一次看到前馈网络逐位置独立作用时，会觉得好像有点奇怪：

前面 attention 好不容易把全句联系起来了，为什么后面又回到每个位置各自处理？

这其实不是退步，而是非常合理的节奏分工。

因为 attention 阶段已经完成了最昂贵、最关键的一步：把上下文相关信息按权重带进来了。到这一步，每个位置手里拿到的已经不是孤立词向量，而是一个上下文化过的表示。

接下来，模型要做的是对这个已经带上下文的信息进行更有表达力的局部变换。前馈网络正适合做这件事。它允许模型在每个位置上进一步形成更复杂的特征组合，而不需要再重复全局交互。

所以“逐位置独立”并不意味着回到孤立处理，而是意味着：

在上下文已经混入之后，对每个位置做更充分的非线性加工。

这一步很像你阅读一句话时，先借上下文把一个词看明白，然后在脑中对这个词当前在句中的角色做更细的内部判断，而不是每一步都重新扫整句。

## 残差连接为什么不是方便训练的小技巧

现在转向第二个问题：深层可训练性。

假设你把很多 attention 和前馈层一层层堆起来。直觉上层数增加应该带来更强表达力，但实际上，深层网络如果结构组织不好，会面临非常现实的困难：信息传递容易变形，梯度传播容易吃力，后层可能反而破坏前面已经学到的有效表示。

残差连接的意义就在这里。

它的核心思想非常朴素：不是让每一层都必须完全重写输入，而是允许这一层在保留原输入通道的同时，只学习一个增量修正。

这件事为什么重要？因为它显著降低了深层训练的难度。模型不必每过一层就从头构造新表示，而可以在原表示基础上逐步修正、逐步增强。

所以残差连接真正带来的，不只是“梯度更好传”这类结果，更深一层看，它改变了层与层之间的工作方式：

一层不再承担“推翻前一层重来”的任务，而更像是在已有表示上做增量加工。

这就让深层堆叠变得现实可行。

## 层归一化为什么和残差常常一起出现

光有残差还不够。因为深层网络里，各层表示的数值分布如果不断漂移，训练过程仍然会变得不稳。层归一化的作用，就是让每个位置在通过某些关键子层之后，表示分布保持在更可控的状态。

你可以把它理解成一种“把当前层输出拉回稳定工作区间”的机制。

这一步的重要性，在深层堆叠里特别明显。因为 attention、前馈、残差叠加之后，如果没有某种规范化，后面层接收到的表示尺度就可能越来越漂，训练就会变得敏感甚至不稳定。

因此，层归一化不是独立炫技，而是在维护整条深层计算链的数值生态。

也正因为如此，残差和归一化常常一起理解最合适。前者负责让信息和梯度有稳定主通路，后者负责让表示分布不要一路漂走。

## 现在把这些组件重新放回 block 里看

到这里，我们可以把完整 Transformer block 的节奏重新看一遍。

先有 self-attention，让每个位置按当前需求和整句其他位置发生信息交换。

然后通过残差与归一化，保证这一步的信息更新不会破坏整体可训练性。

接着进入前馈网络，对每个位置已经上下文化过的表示做更充分的非线性加工。

之后再次通过残差与归一化，让这一轮位置级加工也能稳定地融入主表示通道。

所以一个标准 block 不是“attention 后面顺手接点别的”，而是非常有逻辑的两阶段结构：

先跨位置混合，再位置内加工；
每一步都通过残差与归一化稳住。

这样一来，block 才不是单薄的“谁看谁”，而成为一个真正可堆叠的表示更新单元。

## 用贯穿例子把本课走完

还是那句：

“这只动物没有过马路，因为它太累了。”

假设经过 attention，“它”这个位置已经从“动物”“因为”以及相关事件背景中吸收了一些信息。到这一步，它确实不再是一个薄薄的代词表示了。

但如果停在这里，它仍然只是“混合过信息”。模型还需要进一步把这种混合结果加工成更适合后续层使用的内部特征。也许某些维度需要被放大，也许某些局部模式需要被非线性组合成更稳定的语义角色，这就是前馈网络在做的事。

与此同时，如果这种“attention 更新 + 前馈加工”要重复很多层，没有残差通路，原有表示很容易在反复变换中变得难追踪；没有层归一化，数值分布也容易越来越不稳定。

所以从这个例子里你已经能感觉到：

attention 负责把相关信息找过来；
feed-forward 负责把找来的东西在当前位置真正加工好；
residual 和 layer norm 负责让这一切能一层层稳定积累下去。

## 一点必要的形式化轮廓

现在可以给出最小骨架，但仍然保持“看结构，不背模板”。

一个标准 Transformer block 通常可以被写成两段。

第一段是 attention 子层：

$H_1 = LayerNorm(X + MultiHeadAttention(X))$

第二段是前馈子层：

$H_2 = LayerNorm(H_1 + FFN(H_1))$

这里最重要的不是你记住符号，而是看出结构节奏：

- attention 子层做跨位置交流；
- 残差把原输入保留下来；
- layer norm 稳住表示分布；
- FFN 再做逐位置非线性加工；
- 再一次残差和归一化，让加工结果稳定地回到主表示通道。

这一节奏一旦理解，后面无论看到 Pre-LN 还是 Post-LN 变体，你都不会丢掉主线。

## 本课的局部闭环

我们现在把这节课收一下。

完整 Transformer block 之所以不止有 attention，是因为 attention 主要解决的是跨位置的信息交换，而不是全部表示加工。前馈网络补上了逐位置的非线性表达能力；残差连接让深层堆叠变成增量更新而不是层层推翻；层归一化则维持训练过程中的数值稳定。它们合在一起，不是在给 attention 附加花边，而是在把它真正变成一层可堆叠、可训练、可扩展的结构单元。

如果把本课压成一句最关键的话，那就是：

attention 让位置彼此交流，FFN 让每个位置真正消化这些交流，而残差与归一化让这种更新能够稳定层层累积。
