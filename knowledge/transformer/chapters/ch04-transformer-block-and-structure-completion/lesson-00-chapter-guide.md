# 第 4 章 第 0 课：本章导学与主线说明

第 3 章结束时，Transformer 的核心发动机已经立住了。我们已经知道，同一条序列里的每个位置都可以用自己的 query 去重新阅读整条序列，通过 query-key 匹配、缩放、softmax 和 value 汇总得到新的上下文化表示。到这里，self-attention 的核心已经很清楚了。

但如果你这时停下来，会立刻遇到一个非常现实的问题：

如果 self-attention 已经这么强，为什么完整 Transformer 层里还要加那么多别的东西？

这个问题不能草率回答。因为一旦回答得太浅，位置编码、多头注意力、前馈网络、残差连接、层归一化就会在你的脑子里变成平行零件，像是一堆经验性外挂；可如果这章讲透，你会看到这些部件其实都在补同一个核心发动机的结构缺口。

所以第 4 章的任务，不是简单介绍组件，而是回答：

为什么只有 self-attention 还不够，完整 Transformer block 到底在补什么。

这一章会推进四件事。

第一，我们会解释为什么 self-attention 本身并不天然知道顺序，于是位置编码必须出现。

第二，我们会说明为什么一次注意力往往还不够，模型需要多头并行去捕捉不同类型的关系。

第三，我们会解释为什么 attention 汇总之后，还需要前馈网络对每个位置做进一步的非线性加工。

第四，我们会把残差和归一化放回深层训练的背景里，说明它们不是“稳定训练的小技巧”，而是完整可堆叠结构得以成立的关键条件。

读完这一章之后，你至少应当具备三种稳定能力。

1. 能说明为什么 self-attention 如果没有位置信息，会失去语言顺序感。
2. 能解释多头注意力不是简单重复，而是在让模型并行建不同关系视角。
3. 能把前馈、残差和归一化看成完整 block 的必要补足，而不是附属小组件。

这一章仍然会回收课程里的贯穿句子：

“这只动物没有过马路，因为它太累了。”

但这次我们会用它来问更结构化的问题。比如：如果没有位置编码，模型为什么分不清“谁在前谁在后”？如果只有一个头，模型是否容易把不同关系压在同一种读法里？如果只做注意力混合，不做后续非线性变换，每个位置的表示是不是还不够充分？

请按下面顺序学习，不建议跳读：

- `lesson-00-chapter-guide.md`
- `lesson-01-why-position-encoding-is-not-optional.md`
- `lesson-02-why-multi-head-attention-is-more-than-repetition.md`

这个顺序对应的推进链很明确：

先补顺序感，再补关系视角，再补位置级非线性加工与深层稳定性，最后把完整 Transformer block 的局部闭环锁住。

## 学习提醒

这一章如果某一步让你感觉“好像只是多加了个模块”，先回到两个问题：

- 这个组件是在补 self-attention 的哪一个缺口？
- 它是在解决“表示内容不够”，还是“结构组织不够”，还是“训练稳定性不够”？

只要这两个问题不丢，这一章就不会被你学成部件清单。
