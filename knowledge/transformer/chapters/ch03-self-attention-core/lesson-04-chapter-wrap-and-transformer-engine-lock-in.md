# 第 3 章 第 4 课：本章回收与 Transformer 核心发动机锁定

第 3 章到这里，Transformer 最核心的发动机其实已经真正露出来了。现在最重要的，不是急着往后堆新部件，而是先把这一章已经建立起来的结构视角彻底锁住。因为只要这一步没锁住，后面你很容易把位置编码、多头注意力、前馈网络这些东西看成平行堆件，而看不见它们其实是在围绕同一个核心发动机做补足。

所以这一课的任务很明确：把第 3 章压成一条连续主线，并说明 self-attention 为什么足以被称为 Transformer 的核心发动机，同时又为什么它还不是整辆车。

## 先回到这一章最初的问题

第 3 章最开始问的，是：

在 Transformer 里，前面已经建立好的注意力骨架，究竟怎样被写成 self-attention？

这件事现在已经可以完整回答了。

第一课先把“self”讲清楚了。所谓 self-attention，不是自己只看自己，而是同一条序列内部各位置彼此做注意力。所有 query、key、value 都来自同一组输入表示，因此每个位置都可以用自己的 query 去重新阅读整条序列。

第二课则解释了为什么分数常写成点积。点积不是任意选的乘法技巧，而是在向量层面上度量“当前位置的需求方向”和“候选位置的匹配方向”是否对齐。它既自然，又高效，还非常适合后续矩阵并行化。

第三课进一步说明，这还不够。因为点积分数会受到维度影响，直接送进 softmax 容易让分布过尖，于是需要除以 $\sqrt{d_k}$ 做尺度控制。同时，整条序列中所有位置彼此打分和汇总的过程，也可以被统一写成矩阵运算，这让 self-attention 不只是概念成立，而且能在工程上成为高效核心层。

所以第 3 章的工作，实际上是把“注意力原型”压缩成了“可训练、可稳定、可并行”的 self-attention 标准形式。

## 把第 3 章压缩成一条真正的发动机链

如果我们不用课时标题，而只用一个连续机制链来概括这一章，那么这条链是这样的：

同一条序列上每个位置都有当前表示；
这些表示分别生成各自的 query、key、value；
每个位置用 query 去和整句所有位置的 key 比较；
这些比较形成一张全序列相关性图；
相关性图经过缩放和 softmax，变成权重分配；
权重分配再去汇总整句各位置的 value；
于是整条序列中每个位置都得到一个新的上下文化表示。

这就是 self-attention 的完整发动机过程。

请注意，这里最关键的不是公式写出来了，而是你已经能看见：

self-attention 真正改变的，不是某个局部技巧，而是整条序列中每个位置获得上下文的方式。

以前位置的信息关系更多是沿着顺序链条慢慢传，现在则变成同层内直接建立匹配和汇总。这个变化大到足以让整个序列建模范式改写。

## 为什么说它是发动机，而不是普通模块

在神经网络里，很多部件都可以叫“模块”。但 self-attention 在 Transformer 里更接近发动机。这个说法不是夸张，而是因为它承担了最核心的那件事：

让每个位置根据当前需求，直接从整条序列中动态吸收相关信息。

一旦这件事成立，很多以前很痛苦的问题都会换一种表达方式。长距离依赖不再主要靠长链传输，而可以靠直接匹配；上下文建模不再只是顺序继承，而变成全局相互作用；每个位置的表示也不再是局部静态标签，而会在同层里因为整句其余位置而被重新定义。

这就是为什么我们说 self-attention 是 Transformer 的核心发动机。因为后面别的组件虽然也重要，但它们要么是在给这个发动机补缺口，要么是在让这个发动机更稳定、更丰富、更可堆叠。

## 再把贯穿例子回收到发动机视角

还是那句话：

“这只动物没有过马路，因为它太累了。”

到第 3 章结束时，你已经不只是知道“这句话里代词有上下文依赖”，而是应该能把它放进 self-attention 的标准机制里解释。

“它”这个位置会形成自己的 query，整句所有位置提供 key 和 value。通过点积打分、缩放、softmax 权重化之后，“它”会从整句中得到一份按相关性加权汇总来的新表示。这个新表示不再只是原来那个薄薄的代词位置，而是已经吸收了指代对象、局部事件背景和句内关系的上下文化结果。

更重要的是，这件事不是只有“它”在做。整句所有位置都在同一层里这样做。于是整句话会被整体重写一遍，每个位置都因为其余位置而得到新的定义。

这就是发动机真正启动后的样子。

## 但为什么到这里还不能说“Transformer 已经讲完了”

到这里有一个非常关键的转折必须先讲清楚。

self-attention 虽然已经是核心发动机，但它还不是完整 Transformer 层。原因并不复杂：发动机能够让车动起来，但整辆车能不能跑得稳、能不能转向、能不能适应复杂路况，还需要别的部件配合。

在 self-attention 本身里，至少还埋着几个没有解决彻底的问题。

第一，self-attention 本身并不天然知道顺序。因为它看的是位置与位置之间的相关性，但如果不额外补位置信息，它并不知道“谁在前谁在后”为什么重要。

第二，仅靠一次 value 汇总，模型虽然得到了混合上下文的表示，但这还不等于足够丰富的非线性特征加工。

第三，深层堆叠时，如果没有额外结构保护，训练稳定性会出问题。

所以后面完整 Transformer 层里还必须有位置编码、前馈网络、残差连接和归一化。这些东西不是在另起炉灶，而是在回答 self-attention 还没完全解决的剩余问题。

这一点先看清楚非常重要。因为这样你进入下一章时，就不会把那些组件看成平行补丁，而会知道它们是在围绕同一个核心发动机做结构补足。

## 这一章真正给你建立了哪几层能力

如果不按知识点，而按理解能力来讲，第 3 章至少建立了四层东西。

第一层，是对象关系能力。你已经知道 self-attention 的 self 指的是同一条序列内部各位置彼此做注意力，而不是某种“自己看自己”的字面意思。

第二层，是分数理解能力。你已经知道点积不是装饰性乘法，而是在衡量 query 需求方向与 key 匹配方向的对齐程度。

第三层，是稳定性理解能力。你已经知道缩放不是可有可无的小修补，而是在控制分数尺度，保护 softmax 和训练过程。

第四层，是并行视角能力。你已经知道矩阵化不是为了把公式写漂亮，而是在把整条序列的两两相关性一次并行算出来。

这四层能力一旦形成，你对 self-attention 的理解就已经不只是“会抄公式”，而是真正知道这台发动机在干什么。

## 本章最终收束

如果把第 3 章压缩成一句高层直觉，那么可以这样说：

self-attention 的本质，是让同一条序列中的每个位置都能拿着自己的问题，直接重读整条序列，并把读到的相关内容汇成新的自己。

如果再把这句话压得更技术一点，那就是：

Transformer 的核心发动机，是一个通过 query-key 匹配、缩放、softmax 和 value 汇总来重写整条序列表示的并行机制。

到这里，第 3 章的局部闭环已经完成。下一章开始，我们就不再问“self-attention 怎么工作”，而会问：

既然核心发动机已经有了，为什么完整 Transformer 层还必须有位置编码、多头注意力、前馈网络、残差和归一化？
