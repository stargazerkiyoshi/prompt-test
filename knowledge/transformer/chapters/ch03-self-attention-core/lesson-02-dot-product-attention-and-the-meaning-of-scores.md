# 第 3 章 第 2 课：点积注意力与“分数”到底在表达什么

上一课我们已经把 self-attention 的对象关系讲清楚了。现在可以正式问一个技术上非常核心的问题：

当当前位置的 query 要和整句各位置的 key 做匹配时，这个匹配分数到底怎么来？

在 Transformer 的标准形式里，最常见的答案是：用点积。

很多教材到了这里会直接给你公式，然后说这是 dot-product attention。但如果只停在这个说法上，你往往会有一种不踏实的感觉：为什么偏偏是点积？点积在这里到底是数学技巧，还是有某种直观意义？

所以这一课的任务，不是只让你记住一个公式，而是把“点积为什么在这里成立”真正讲清楚。

## 先从最朴素的问题出发：相关性需要一种怎样的比较

当当前位置拿着 query 去看整句中其他位置的 key 时，它要回答的问题其实非常具体：

这些候选位置里，谁更像当前我需要的那个信息来源？

这听起来像语义判断，但一旦落到模型里，它就需要一种可计算的比较方式。而这种比较方式至少要满足三点要求。

第一，它要能区分“更相关”和“更不相关”，也就是给出强弱差异。

第二，它要适合在向量表示之间工作，因为 query 和 key 都已经不是文字标签，而是高维向量。

第三，它要足够高效，因为 self-attention 里不是只比一次，而是整条序列上所有位置彼此都要大量比较。

在这些要求之下，点积会显得非常自然。因为它本来就是衡量两个向量对齐程度的一种方式。两个向量越朝着相似方向，点积通常越大；越不一致，点积就越小，甚至可能为负。

所以如果你先不把它想成复杂公式，而只把它理解成一种“向量层面的相合程度”，那么点积出现在这里就一点也不突兀。

## 点积在这里的直觉意义：当前需求和候选特征是否对得上

我们已经知道，query 表达的是当前位置当前在找什么，key 表达的是一个候选位置在匹配层面上“我像不像你要找的东西”。

那两者做点积，直觉上就在问：

当前这个需求方向，和这个候选位置可被匹配的特征方向，对得上多少？

这个说法很重要，因为它把点积从纯数学运算拉回了机制意义。

还是那个句子：

“这只动物没有过马路，因为它太累了。”

当当前位置是“它”时，它形成的 query 可以理解成一种“我正在找合适先行对象”的需求方向。

这时候，“动物”这个位置的 key 可能就会在某些维度上更像一个实体候选，于是和 query 的对齐程度更高，点积更大。

而“没有”这个位置的 key 更偏向极性信息，不是代词解析的主要目标，于是和这个 query 的对齐程度较弱，点积就可能更小。

请注意，这里真正重要的不是你能不能把每一维语义解释出来，而是你已经知道：

点积不是凭空做乘法，它是在度量 query 的需求方向与 key 的候选方向有多对齐。

这一步一旦看清，后面公式就开始“有意思”了，而不再只是“有样子”。

## 为什么不一定非要点积，但点积为什么特别合适

这里需要讲得严格一点。理论上，相关性比较并不只有点积这一条路。你完全可以设计更复杂的打分函数，比如带额外参数的小网络，或者别的相似度形式。

所以我们不能说“只有点积才行”。

但在 Transformer 里，点积之所以成为标准核心选择，是因为它恰好很好地平衡了三件事：

- 它表达力够用，能够捕捉向量对齐程度；
- 它计算高效，适合大规模并行矩阵运算；
- 它结构简单，容易训练，也容易和后面的 softmax 权重化过程衔接。

所以这里更准确的说法不是“点积是唯一正确的”，而是：

在 self-attention 这个任务里，点积是一种非常自然、非常高效、也非常适配矩阵并行化的打分方式。

这句话比单纯背“用了点积”更有解释力。

## 把单个位置的打分走一遍：当前位置如何对整句形成一排分数

现在我们把单个位置的计算过程讲完整。

假设当前位置是第 $i$ 个位置，它有自己的 query，记作 $q_i$。整句所有位置都有 key，分别记作 $k_1, k_2, \dots, k_n$。

那第 $i$ 个位置要做的事情就是：

把自己的 $q_i$ 分别和整句每个位置的 $k_j$ 做点积，得到一组分数。

也就是说，它不会只得到一个分数，而是得到一整排分数：

- 它和位置 1 有多匹配；
- 它和位置 2 有多匹配；
- ……
- 它和位置 n 有多匹配。

这排分数就像当前位置对整句做出的一次相关性扫描图。

如果你还停留在第 2 章的直觉层，这里就等价于：

当前位置拿着自己的“问题”，把整句候选位置逐个扫一遍，看谁更像答案来源。

所以单个位置的点积打分，本质上是在生成当前位置的“全句相关性视图”。

## 为什么这里的分数不是语义解释本身，而是后续分配的原材料

还有一个容易混淆的地方也要压住。

当我们说“动物”的分数更高时，很容易不自觉地把这个分数本身当成语义结论，好像模型已经完成了理解。其实不是。这个分数只是后续权重分配的原材料。

它的作用不是直接告诉你整句含义是什么，而是告诉当前位置：

在后面的 value 汇总里，哪些位置应该拿到更大份额，哪些位置应该被弱化。

所以你可以把点积分数理解成一种“预分配信号”。它本身还不是最终表示，也不是最终输出，它只是决定信息流该怎么走。

这一层看清非常重要，因为后面你才不会把 attention map 和真正的表示更新混为一谈。

## 现在把最小公式正式写出来

到了这里，公式已经不会显得突然了。

单个位置 $i$ 对位置 $j$ 的原始匹配分数，可以写成：

$score_{ij} = q_i \cdot k_j$

这里的点积，就是我们前面一直在解释的“当前需求方向”和“候选匹配方向”的对齐程度。

如果把第 $i$ 个位置对整句所有位置的比较都写出来，你就会得到一整排分数：

$q_i \cdot k_1,\ q_i \cdot k_2,\ \dots,\ q_i \cdot k_n$

这就是当前位置对整句的原始注意力打分。

请注意，这里我们还没有做 softmax，也还没有去汇总 value。现在只是完成了“相关性打分”这一步。

## 为什么点积一大，softmax 之后差异会变得更明显

虽然完整的 softmax 作用在上一章已经讲过了，但这里有必要提醒你一个和点积直接相关的现象。

点积打出来的分数如果差异较大，那么经过 softmax 后，权重差异往往会被进一步拉开。也就是说，打分阶段看起来只是略高一些，到了权重分配阶段可能就会明显更突出。

这一点有好处，因为它能让模型更清楚地聚焦当前最 relevant 的位置。但它也埋下了一个后面必须处理的问题：如果分数规模太大，softmax 可能会变得过于尖锐，训练就会受影响。

这正是下一课里“为什么要缩放”会出现的原因。

所以你现在可以先预告性地记住：

点积很自然，但点积的数值规模不能完全不管。

## 把本课走到局部闭环

现在把这一课收一下。

self-attention 里，query 和 key 的匹配分数通常用点积来计算。这个选择不是偶然技巧，而是因为点积很自然地衡量了“当前位置当前需求方向”和“候选位置匹配方向”的对齐程度。同时，它计算高效、结构简单、适合大规模并行化。

因此，点积打分在 self-attention 里真正做的事情，不是直接给出语义答案，而是为后续权重分配提供原材料，让当前位置知道整句中哪些位置更值得被吸收进自己的新表示。

如果把本课压成一句最关键的话，那就是：

点积不是在做装饰性乘法，而是在把“相关不相关”压成可计算、可分配的信息流信号。
