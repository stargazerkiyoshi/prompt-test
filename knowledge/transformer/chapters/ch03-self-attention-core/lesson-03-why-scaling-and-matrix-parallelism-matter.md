# 第 3 章 第 3 课：为什么要缩放，为什么要矩阵化并行

上一课我们已经把 self-attention 的原始打分讲清楚了：当前位置的 query 和整句各位置的 key 做点积，得到一组相关性分数。到这里，很多人会觉得机制已经差不多完整了，接下来无非就是 softmax 再乘 value。

但 Transformer 的标准形式里还有两件事绝不能一笔带过：

第一，为什么点积之后还要做缩放，也就是除以 $\sqrt{d_k}$。

第二，为什么这整套机制能被写成矩阵运算，一次并行算完整条序列。

这两件事看上去一个偏数值细节，一个偏工程组织，但实际上都非常核心。前者决定训练过程会不会因为分数分布失控而变得难学，后者则决定 self-attention 这种想法是否真的能成为大规模可用的核心架构，而不只是一个漂亮但慢得不现实的概念。

## 先解决第一个问题：为什么点积分数不能放着不管

我们先回忆一下上一课最后留下的提醒。

如果 query 和 key 的维度变大，它们的点积结果通常也会在数值上变得更大、更分散。直觉上很好理解，因为点积是在把许多维度上的乘积加起来，维度越多，可累积出来的量往往就越大。

这件事一旦进入 softmax，会发生什么？

softmax 会把较大的分数进一步放大成明显更高的权重，把较小的分数压得更低。如果原始分数已经因为维度高而变得很大，那么 softmax 后的分布就很容易变得过尖。也就是说，某几个位置会突然拿到极高权重，其他位置几乎全部被压到接近零。

乍一看你可能会觉得，这不是很好吗？模型更聚焦了。

问题在于，聚焦并不总是越尖越好。过尖的 softmax 会让梯度信号变差，让模型更难在训练中细腻地调整各位置权重。很多时候模型还没学稳，就已经被过于自信的打分推到了一个很难微调的状态。

所以这里的问题不是“点积能不能用”，而是：

点积的数值规模如果随着维度变大而失控，就会干扰后续 softmax 的工作状态。

这就是缩放必须出现的地方。

## 为什么除以 $\sqrt{d_k}$ 会显得自然

现在我们要讲清楚的，不只是“论文里这么写”，而是为什么这个缩放看起来合理。

如果 key 的维度是 $d_k$，那么点积大致可以看成很多维乘积的累加。维度越大，累加后的值波动范围通常也会变大。为了不让这种规模膨胀直接冲进 softmax，我们需要在进入 softmax 之前，把分数压回一个更稳定的量级。

除以 $\sqrt{d_k}$ 正是这样一种标准化思路。它不是在改变“谁比谁更相关”的相对排序，而是在控制这些分数的整体尺度。

所以你可以把缩放理解成一句非常朴素的话：

既然维度变大天然会放大点积数值，那就先把这个放大量做一个有规律的抵消，让后续 softmax 工作在更稳定的区间里。

这一步的意义，不是让公式更漂亮，而是让训练过程更可控。

因此，scaled dot-product attention 里的 “scaled” 绝不是附属装饰，而是和数值稳定性直接绑定的。

## 缩放并没有改变机制本质，只是在保护机制可训练

这里还要特别指出一件事，不然容易把缩放理解错。

缩放不是在修改注意力的核心逻辑。核心逻辑仍然是：

query 和 key 比较相关性，分数经过 softmax 变成权重，再去汇总 value。

缩放做的事情只是：在“比较相关性”到“进入 softmax”之间，加上一层尺度控制，避免分数因为维度效应变得过大。

也就是说，它不是换了一种注意力，而是在保护原本这套注意力能够稳定训练。

这一点非常像很多优秀工程设计中的共同特征：真正厉害的结构，不只是有主意，还知道怎么让这个主意在大规模训练里不崩。

## 现在转向第二个问题：为什么矩阵化并行不是可有可无

到这里，单个位置的机制已经很清楚了。一个位置拿自己的 query 去和整句所有位置的 key 比较，得到权重，再汇总 value。

但如果你只停在这个“单个位置逐个比较”的图景里，Transformer 的威力仍然只看了一半。

因为 self-attention 真正强的地方，不只是它允许直接建联系，还在于：

整条序列中所有位置都可以同时做这件事，而且可以被组织成高度并行的矩阵运算。

这个“同时”非常关键。它意味着模型不是先处理第 1 个词，再等结果传给第 2 个词，再等第 2 个传给第 3 个，而是同一层里所有位置都能一起发起自己的 query，对整句做匹配和汇总。

这正是 Transformer 和很多循环式处理方式在计算组织上的重大分水岭。

## 从单个位置过渡到整条序列：为什么自然会出现 Q、K、V 矩阵

假设一句话有 $n$ 个位置。每个位置都有一个表示，每个表示都可以生成自己的 query、key、value。

那如果你把整句所有位置的 query 竖着堆起来，就得到一个 query 矩阵 $Q$。

同样地，把所有 key 堆起来，得到 $K$；把所有 value 堆起来，得到 $V$。

这一步非常自然。因为我们并不是突然为了写矩阵而写矩阵，而是在说：

既然整句每个位置都各自有 query、key、value，那就把它们一次性打包起来一起算。

一旦这样打包，原本“每个位置分别和全句比较”的大量点积运算，就可以一次写成：

$QK^T$

这一下子就把整条序列两两之间的匹配分数全算出来了。

请注意这里发生了什么。$QK^T$ 的结果不是一个单值，而是一个分数矩阵。这个矩阵里的第 $i$ 行第 $j$ 列，恰好就是位置 $i$ 的 query 和位置 $j$ 的 key 的匹配分数。

也就是说，整张矩阵本质上就是：

整条序列上所有位置对所有位置的相关性地图。

## 为什么这个矩阵视角一旦成立，self-attention 就从“好想法”变成“强架构”

如果注意力只能逐位置串行做，虽然概念上也有意义，但它的工程优势会被大幅削弱。真正让 Transformer 爆发出的，不只是“位置可以直接相互看”，而是“这种相互看可以被高度并行化地同时算完”。

一旦 $QK^T$ 可以一次得到全序列相关性矩阵，再接 softmax，再乘上 $V$，整个 self-attention 层就能在现代硬件上高效运行。

所以矩阵化并行不是附带技巧，而是 Transformer 之所以能在实践中统治大量序列任务的关键条件之一。

你可以这样理解：

第 1 章解决的是为什么要直接访问上下文；
第 2 章解决的是这种访问怎样变成注意力骨架；
第 3 章到这里，则进一步说明，这套骨架不仅合理，而且能被非常适合并行硬件的方式组织起来。

这三步连起来，Transformer 的核心价值才真正完整。

## 现在可以把标准骨架公式写完整

到这里，标准 self-attention 公式就不会再只是一个硬记模板了。

给定输入表示矩阵 $X$，我们先通过线性变换得到：

$Q = XW_Q,\quad K = XW_K,\quad V = XW_V$

然后，原始分数矩阵是：

$QK^T$

接着做缩放：

$\frac{QK^T}{\sqrt{d_k}}$

再经过 softmax，得到每个位置对所有位置的权重分配：

$softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)$

最后乘上 $V$，得到整条序列新的上下文化表示：

$Attention(Q,K,V) = softmax\left(\frac{QK^T}{\sqrt{d_k}}\right)V$

请注意，这个公式现在已经有完整意义了：

- $QK^T$ 给出全序列相关性分数图；
- 除以 $\sqrt{d_k}$ 控制分数尺度；
- softmax 把分数变成权重分配；
- 乘 $V$ 把权重落成内容汇总，输出整条序列的新表示。

如果没有前面几课的铺垫，这个公式只是形式；有了铺垫之后，它其实就是一个高度压缩的机制叙述。

## 用贯穿例子最后再对一遍

还是回到那句话：

“这只动物没有过马路，因为它太累了。”

在 self-attention 里，这句话每个位置都会先生成自己的 query、key、value。

当“它”所在位置对应的那一行 query 去乘整句所有位置的 key 时，它就得到一整排分数。经过缩放和 softmax 后，这一排分数会变成“它”对整句各位置的权重分配。然后这组权重去汇总整句各位置的 value，得到一个新的、更上下文化的“它”的表示。

与此同时，其他位置也都在做同样的事。比如“动物”也在问自己的问题，“因为”也在问自己的问题。所有这些更新在同一层里一起完成。

所以 self-attention 的结果，不是只把一个词解释清楚，而是把整句所有位置都重新放回彼此关系之中，统一重写了一遍。

## 本课的局部闭环

我们现在把这节课收一下。

点积本身很自然，因为它能衡量 query 和 key 的对齐程度；但维度上升会让点积分数规模变大，于是必须通过除以 $\sqrt{d_k}$ 来稳定进入 softmax 之前的数值尺度。与此同时，self-attention 的真正力量不只在单个位置怎样工作，而在于整条序列的所有位置都能同时做这件事，并被组织成矩阵并行运算。于是，Transformer 的核心机制既有结构上的合理性，又有计算上的现实可行性。

如果把本课压成一句最重要的话，那就是：

scaled dot-product attention 不只是一个公式，它是“相关性判断 + 数值稳定 + 全序列并行”三件事被压在一起的结果。
