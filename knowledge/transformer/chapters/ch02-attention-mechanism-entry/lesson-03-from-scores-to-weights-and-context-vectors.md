# 第 2 章 第 3 课：从分数到权重，再到上下文化向量

上一课我们已经把注意力拆成了一套匹配系统：当前位置提出 query，其他位置用 key 参与匹配，真正被取来汇总的是 value。现在问题就变得非常具体了：

如果一个 query 和所有 key 都比较过了，接下来该怎么办？

这个问题就是注意力机制从“概念原型”走向“可计算机制”的关键一步。因为匹配之后，模型不能只停在一句模糊判断上，比如“这个位置更相关，那个位置稍微弱一点”。模型必须把这种相关性变成可以连续调节、可参与训练、最终能更新表示的数值结构。

所以这节课要解决的是一条完整链：

匹配怎么变成分数；
分数怎么变成权重；
权重又怎样把多个 value 汇总成当前位置的新表示。

## 先解决第一个问题：为什么相关性要落成分数

在自然语言里，我们习惯说“这个词跟那个词关系更大”。但模型不能停在这种语言判断上，因为它后面还要计算、还要反向训练、还要让不同位置的影响被连续调节。

于是，一个很自然的第一步就是：

让 query 和各个 key 的匹配结果，先落成一组数值分数。

这些分数不必一开始就被神秘化。你可以先把它理解成“当前位置觉得各个候选位置有多相关”的粗量化。

比如当模型处理“它”时，前文“动物”的匹配分数可能比较高，“没有”可能比较低，“因为”可能中等偏低。这里重要的不是具体数字是多少，而是模型终于把“谁更 relevant”这件事压成了可比较的量。

一旦有了分数，模型就能做两件以前做不到的事。

第一，它可以在多个候选位置之间建立明确强弱顺序，而不只是模糊地区分“相关”与“不相关”。

第二，它可以把这种强弱关系继续传递给后面的内容汇总，而不是停在一个离散判断上。

这时候你已经能看到，注意力和传统的“硬匹配”很不一样。它不是只问“是不是这个位置”，而是在问“这些位置分别有多大程度值得被吸收”。

## 为什么通常不直接做硬选择

如果只是为了找最相关的位置，最直观的办法当然是：谁分数最高，就选谁。那为什么注意力通常不这么做，而是继续把分数变成一组权重？

因为语言理解里，当前位置需要的上下文往往不是一个绝对单点，而是一种有主有次的组合。

还是那个句子：

“这只动物没有过马路，因为它太累了。”

当模型处理“它”时，“动物”当然可能是最强候选。但当前位置真正需要的表示，不一定只靠“动物”单独完成。它还可能需要一点点事件背景、句法位置、逻辑结构。它的主要信息来源也许有冠军，但通常不止冠军。

这也是人类理解的常态。你在理解一个代词时，脑中最先想到一个对象，但周边背景并不会完全清零。语言意义常常是主线索加辅助线索共同塑造出来的。

所以硬选择太脆。它会让模型把注意力机制退化成“只认一个位置”，而失去对复杂上下文细粒度整合的能力。

因此，注意力更常见的做法是：

不把分数变成一个唯一获胜者，而是变成一组权重，让多个位置以不同强度参与。

这个决定非常关键。它把注意力从“位置选择器”变成了“上下文混合器”。

## 从分数到权重：为什么需要一种归一化后的相对分配

现在问题继续推进。既然我们要让多个位置按不同程度参与，那这些分数怎样才能真正变成可用权重？

这里至少要满足两个要求。

第一个要求是可比较。不同位置的影响应该落在同一套刻度上，不然我们没法稳定地说谁贡献更大。

第二个要求是可分配。最终这些影响要进入一次汇总，因此最好能形成一种“总量被分给各位置”的结构。

于是，非常自然的想法就是：把原始分数再变成一组归一化后的权重。这样每个位置都会得到一个非负的重要性比例，总体加起来形成一次完整分配。

在注意力机制里，这一步通常会通过 softmax 完成。你现在先不用把它当成一个必须背熟的函数名，而要先理解它在这里扮演的角色：

它把“粗糙匹配分数”变成“可用于分配影响力的权重”。

这一步的意义，不是数学洁癖，而是把相关性判断推进成内容汇总的接口。

也就是说，到这里为止，query-key 匹配得到的不再只是排序信息，而是已经变成“每个位置的 value 应该以多大比例参与当前位置更新”。

## 为什么 softmax 这种形式会特别合适

虽然我们不打算这节课把 softmax 讲成纯数学章节，但有一层动机必须说清。

softmax 的好处在于，它保留了相对强弱，同时把结果压成一组总和为 1 的权重。于是你就可以把它理解成：

当前位置把自己的注意资源，按不同比例分给了不同候选位置。

这个解释非常有帮助。因为它让“权重”不再只是数字，而像是一种资源分配图。

当某个位置分数显著更高时，它会拿到更大份额。

当多个位置都比较相关时，资源会在它们之间分开分配。

当某个位置几乎不相关时，它分到的份额就会很小。

于是，softmax 把一个很抽象的匹配过程变成了一个很直观的结果：当前这个位置正在以什么样的比例，向上下文中的其他位置索取信息。

## 最关键的一步：权重不是终点，Value 汇总才是终点

现在来到本课真正的核心。

很多人在第一次学注意力时，会把注意力权重本身当成终点，觉得“模型知道该看谁”好像事情就结束了。其实不是。权重只是中间桥梁，真正的目标是：

用这组权重去加权汇总各位置的 value，构造当前位置新的表示。

为什么一定要强调这一点？因为注意力的目的不是解释世界，而是更新表示。它不是为了画一张好看的热力图，而是为了让模型在后续层里拿到一个更有用的上下文化向量。

这时候就能看出 value 的地位了。每个位置把自己的内容表示拿出来，当前位置再按照前面得到的权重，对这些内容表示做加权求和，最后形成一个新的 context vector。这个向量就是当前位置这轮“读上下文”之后真正获得的结果。

你可以把它理解成：

当前位置去周围收集信息，回来之后生成了一份新的、已经吸收上下文的自己。

这就是为什么注意力输出不是简单复制某个词，而是一种重建。

## 把贯穿例子完整走完：从“它”的查询，到新的表示生成

现在我们把那句核心例子按完整链条走一遍。

句子是：

“这只动物没有过马路，因为它太累了。”

当模型走到“它”时，当前位置形成一个 query，表达“我要找到当前代词所依赖的对象与上下文”。

前文各位置拿出自己的 key，与这个 query 比较，得到一组分数。

“动物”的分数比较高，因为它很像当前位置要找的先行对象。

“过马路”可能不是主候选，但提供事件背景，分数未必为零。

“因为”在逻辑结构上有关联，也可能保留少量影响。

“没有”对代词指向帮助较小，所以权重可能更弱。

接着，这些分数被转成一组权重。于是模型不是简单宣布“动物赢了”，而是更细腻地说：

当前位置的更新，主要依赖“动物”的 value，同时少量吸收其他相关位置的 value。

最后，所有这些 value 按权重汇总成一个新的上下文化向量。这个向量不再只是原来那个薄薄的“它”，而是一个已经带着指代线索、局部事件背景和句内结构关系的新表示。

这一步就是注意力机制真正完成工作的地方。

## 现在可以把最小公式轮廓放出来

到这里，公式已经不会显得突兀了。我们只放最小骨架，不在这节课里做过细技术展开。

对于当前位置 $i$，它与各个位置 $j$ 的匹配分数可以写成某种

$score(q_i, k_j)$

然后这些分数经过 softmax，变成权重

$\alpha_{ij}$

最后，当前位置新的上下文化表示可以写成

$c_i = \sum_j \alpha_{ij} v_j$

请注意，这里最值得你抓住的不是公式长相，而是它压缩出来的三层关系：

- 分数负责表达相关性强弱；
- 权重负责把相关性变成资源分配；
- 加权和负责把资源分配落成新的表示。

只要这三层关系是清楚的，后面真正进入 dot-product attention 时，你就不会被公式表面压住。

## 本课的局部闭环

现在我们把这节课收一下。

上一课我们已经知道，注意力不是平均看上下文，而是有偏向地看。这一课把这个偏向真正写成了一条可计算链。query 和 key 的匹配先变成分数，分数再被归一化成权重，权重最后调度各个 value 参与汇总，构造出当前位置新的上下文化向量。

到这里，注意力已经不再只是概念图景，而开始成为一个真正可执行的机制框架。

如果把本课压成一句最重要的话，那就是：

注意力的核心不是“找到谁最重要”，而是“让相关位置按不同权重共同重写当前位置的表示”。
