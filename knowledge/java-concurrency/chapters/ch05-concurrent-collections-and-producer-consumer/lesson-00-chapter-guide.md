# 第 5 章 第 0 课：本章导学与主线说明

这一章开始讨论：当多个线程共享的已经不是一个简单字段，而是一批集合、映射和队列时，普通数据结构为什么很容易失效？以及 Java 为什么要提供并发容器和阻塞队列。

本章重点是把“共享状态保护”推进成“共享数据结构协作”。你会看到 `ConcurrentHashMap`、`CopyOnWriteArrayList`、`BlockingQueue` 和生产者-消费者模型分别在补什么缺口。

本章课时顺序：

- `lesson-00-chapter-guide.md`
- 第 1 课“为什么普通集合在并发下会出问题”
- 第 2 课“`ConcurrentHashMap` 与读写并发直觉”
- 第 3 课“`BlockingQueue` 与生产者-消费者”
- 第 4 课“本章回收：订单事件流怎样进队列、出队列、被消费”

读完本章后，你应该能回答三个问题：

1. 为什么 `HashMap`、`ArrayList` 不能简单拿来做并发共享容器。
2. 为什么有些场景适合并发映射，有些场景适合复制写容器。
3. 为什么队列不仅是数据结构，也是线程协作协议的一部分。
