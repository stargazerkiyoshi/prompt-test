# 第 3 章 第 0 课：本章导学与主线说明

这一章开始正式进入互斥与同步。上一章回答了“线程之间怎么看见彼此”，这一章要回答：

当共享状态上的一段操作本来就不能被拆开时，Java 怎样让多个线程受控地进入、退出、等待和唤醒？

本章会从 `synchronized` 起步，再走到 `ReentrantLock`、`Condition` 和 `happens-before` 在锁语义里的落点。

本章课时顺序：

- `lesson-00-chapter-guide.md`
- 第 1 课“为什么原子性常常需要互斥”
- 第 2 课“`synchronized` 的语义与适用边界”
- 第 3 课“`Lock` 与 `Condition` 在补什么”
- 第 4 课“本章回收：从库存扣减到等待通知模型”

读完本章后，你应该能回答三个问题：

1. 为什么很多复合操作不是靠 `volatile`，而是靠互斥保护。
2. `synchronized` 和 `ReentrantLock` 分别适合什么场景。
3. 等待-通知、条件队列和锁释放之间为什么必须一起理解。
