# 第 4 章 第 0 课：本章导学与主线说明

这一章把视角从“共享状态保护”推进到“任务怎样被组织和调度”。如果前面几章解决的是线程之间如何安全共享状态，那这一章解决的就是：

真实工程里，为什么并发不应该等于“随手开线程”，以及任务怎样被受控地提交、执行、等待、组合和回收？

本章会进入 `ExecutorService`、`Callable`、`Future`、线程池参数和 `CompletableFuture` 的任务编排能力。

本章课时顺序：

- `lesson-00-chapter-guide.md`
- 第 1 课“线程创建成本与线程池的必要性”
- 第 2 课“`ExecutorService`、`Callable`、`Future` 的任务模型”
- 第 3 课“`CompletableFuture` 如何表达并发编排”
- 第 4 课“本章回收：订单流水线怎样拆成受控异步任务”

读完本章后，你应该能回答三个问题：

1. 为什么线程池不是性能装饰，而是资源控制模型。
2. 为什么“任务”比“线程”更适合作为业务并发的组织单位。
3. 怎样表达串行、并行、依赖和汇合，而不是只会提交一个异步任务。
