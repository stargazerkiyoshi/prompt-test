# 第 2 章 第 0 课：本章导学与主线说明

这一章的任务，是把第 1 章看到的并发混乱压缩成一套统一语义：Java 内存模型（Java Memory Model, JMM）。

如果说上一章是在问“并发为什么会把程序搞乱”，那这一章就是在问：

Java 准备用什么规则说明线程之间怎么看见彼此、按什么顺序约束彼此、以及 `volatile` 为什么重要但不万能？

本章会先从主内存、工作内存和可见性直觉进入，再解释 `volatile` 到底保证什么、不保证什么，最后把问题推进到重排序与 `happens-before`。

本章课时顺序：

- `lesson-00-chapter-guide.md`
- 第 1 课“主内存、工作内存与可见性直觉”
- 第 2 课“`volatile` 到底保证什么，不保证什么”
- 第 3 课“重排序与 `happens-before` 的入口”
- 第 4 课“用库存开关与停止标志走完一个完整例子”

读完本章后，你应该能回答三个问题：

1. 为什么一个线程改了值，另一个线程不一定立刻看见。
2. 为什么 `volatile` 能解决一些问题，却解决不了库存扣减这种复合更新问题。
3. 为什么写并发代码时，真正重要的不是“源码顺序看起来合理”，而是有没有建立可靠的跨线程先后关系。
