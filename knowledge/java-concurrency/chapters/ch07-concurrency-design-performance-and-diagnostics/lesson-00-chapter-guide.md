# 第 7 章 第 0 课：本章导学与主线说明

最后一章不再主要新增底层工具，而是把前面所有内容带回工程现场。真正的并发能力，不只是“知道 API”，还包括：

会做工具选择、会判断性能代价、会定位死锁和堆积、会在代码审查里提前看出风险。

本章会讨论线程池参数、锁竞争、上下文切换、死锁、饥饿、活锁、压测和并发代码审查要点。

本章课时顺序：

- `lesson-00-chapter-guide.md`
- 第 1 课“线程池、队列与吞吐-延迟权衡”
- 第 2 课“死锁、饥饿、活锁与常见排障思路”
- 第 3 课“并发代码审查与测试策略”
- 第 4 课“总复盘：把整门课压回一张工具与问题映射图”

读完本章后，你应该能回答三个问题：

1. 为什么有些并发问题明明逻辑正确，却在线上跑得很差。
2. 死锁、饥饿和活锁为什么不能混为一谈。
3. 并发代码为什么必须比普通业务代码更强调测试、观测和审查。
