# 第1章第1课 从“数”到“向量”

这一课先解决一个起点问题：线性代数到底在研究什么对象，为什么课程不是从矩阵开始，而是从向量开始。

如果这个问题没站稳，后面很多内容都会变成技巧堆积。你会看到矩阵乘法、消元、特征值这些东西，却不清楚它们到底在围绕什么转。第 1 课的任务，就是先把“对象”立住。

## 一、当前在解决什么问题

普通代数最熟悉的对象是数。数很适合表示只有大小的量，比如价格、温度、质量、件数。可是一旦我们处理的是位移、速度、力、图像中的位置变化，或者更一般地说，处理那些既能叠加又能按倍数缩放的对象时，单个数就不够了。

这里真正的问题不是“如何记一个点”，而是“如何描述一种可以组合的效果”。

比如：

- 从原点向右走 2 格、向上走 1 格，这不是一个单纯的大小，而是一个方向性的变化。
- 一股力和另一股力共同作用时，它们会叠加。
- 一段速度如果加倍，本质上是在保持方向的同时改变强度。

这些例子都在提示同一件事：我们需要一种对象，能够同时容纳“大小”和“方向”，并且允许我们做两种最基本的操作：

1. 把两个效果叠加起来。
2. 把一个效果放大、缩小，甚至反向。

这就是向量出现的动机。

## 二、先建立概念与关系框架

现在先不要急着记定义，先把关系框架搭起来。

在线性代数里，最基础的不是某种图形，也不是某套公式，而是一类满足共同规则的对象。它们的共同点在于：

- 可以相加。
- 可以数乘。
- 加法和数乘之间有稳定的配合关系。

因此，“向量”最核心的身份，不是“箭头”这张图，而是“可加、可数乘的对象”。

高中阶段，先把向量看成平面或空间中的箭头是最自然的入口。这样做有两个好处。第一，方向感和大小感都能直接看见；第二，加法和数乘都可以画出来。

但大学阶段要立刻往前走一步：箭头只是向量的一种表现，不是向量的全部本质。以后你会看到，函数可以当向量，数列可以当向量，多项式也可以当向量。它们未必像箭头，但只要保留了“相加”和“数乘”的结构，就能进入线性代数。

所以这一课先建立三层认识：

第一层，向量可以先直观地看成有方向的量。  
第二层，向量之间最重要的关系不是“长什么样”，而是“能否相加、怎样数乘”。  
第三层，线性代数研究的是这类结构，而不是某一种具体外形。

这三层关系如果不分清，后面就容易把“图形直觉”误当成“概念本体”。

## 三、从直觉例子走到运算规则

先看最熟悉的平面情形。设有两个位移：

```text
u = (2, 1), v = (1, 3)
```

你可以把它们理解为：

- `u` 表示向右走 2、向上走 1；
- `v` 表示向右走 1、向上走 3。

如果先做 `u`，再做 `v`，总效果就是

```text
u + v = (3, 4)
```

这说明向量加法在表达“效果叠加”。

如果把 `u` 放大 3 倍，那么得到

```text
3u = (6, 3)
```

如果取反向，则有

```text
-u = (-2, -1)
```

这说明数乘在表达“保持结构方向并调整强度”，负号则意味着方向反转。

到这里，向量的两个最基本运算已经出来了。它们不是人为规定出来的技巧，而是从现实问题里直接抽出来的共同结构。

## 四、必要形式化：把直觉收紧成定义

现在再进入必要形式化。为了让后面的推导可追溯，我们先把讨论放在 `R^n` 里，也就是 n 维实数向量空间。

一个向量记作

```text
x = (x1, x2, ..., xn)
```

其中每个分量都是实数。

对于两个向量

```text
x = (x1, x2, ..., xn), y = (y1, y2, ..., yn)
```

定义它们的加法为

```text
x + y = (x1 + y1, x2 + y2, ..., xn + yn)
```

对任意实数 `a`，定义数乘为

```text
ax = (ax1, ax2, ..., axn)
```

这里的 `a` 叫标量，作用是调节向量的倍数。

这个形式化看上去只是“逐分量运算”，但它真正澄清了两件事：

第一，向量不是随便拼起来的数字串，它们必须服从统一运算规则。  
第二，后面所有更复杂的对象，只要也保留这种规则，就可以被放到同样框架下研究。

这也是为什么大学线里会说：`R^n` 不是全部线性代数，只是最基本、最可见的样板。

## 五、概念之间的关系现在是什么

到这里，可以把这一课得到的关系框架压缩成一句更清楚的话：

数适合描述单一大小，向量适合描述可叠加、可缩放的多分量对象；线性代数研究的第一步，就是确认我们处理的对象是否具备这种结构。

于是，接下来会自然出现两个新问题：

第一个问题是：如果手里有几个基本向量，我们能通过加法和数乘造出哪些别的向量？  
第二个问题是：如果一个向量已经给定，怎样判断它能不能由另外几个向量组合出来？

第一个问题会把我们带到“线性组合与张成”。  
第二个问题最终会把我们带到“线性方程组与矩阵”。

换句话说，第 1 课还没有进入大规模计算，但它已经把后面整门课的两条主线都点亮了。

## 六、一个高中视角与一个大学视角

为了兼顾两个层级，这里把同一内容压成两个视角。

高中视角下，你现在先抓住这句话：向量就是既有大小又有方向、并且可以相加和数乘的量。  
大学视角下，你要再多抓一句：向量未必真的是箭头，关键不在外形，而在它是否保留了线性运算结构。

这两句话不是互相冲突，而是同一个概念的两个层次。高中视角提供入口，大学视角提供推广。

## 七、本课小结

这一课解决了一个根本定位问题：线性代数的起点不是矩阵技巧，而是向量这种“可叠加、可缩放”的对象。

我们先从现实中的位移、力、速度进入，看见为什么单个数不够；再建立“向量的本质在于加法和数乘”这一关系框架；最后用 `R^n` 中的分量表达把这种直觉收紧成可推导的形式。

本课得到的核心结果是三点：

- 向量比数多出的，不只是“有方向”，更是“有可组合的结构”。
- 向量加法表达效果叠加，数乘表达强度调整。
- 线性代数研究的不是某一种图形，而是一类满足线性运算规则的对象。

下一课之所以必要，是因为仅仅知道“向量可以加、可以数乘”还不够。我们更关心的是：给定几个向量，它们到底能生成多大的范围。这就进入“线性组合与张成”，也就是第一章真正开始展开结构的地方。
