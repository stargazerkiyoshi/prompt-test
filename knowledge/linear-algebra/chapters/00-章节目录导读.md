# 章节目录说明

本目录按“每一章一个独立目录”组织。

## 目录结构约定

- 每章目录下先放“第 0 课”导学文件，命名为 `lesson-00-*.md`。
- 再放该章正式课时讲解文件，命名为 `lesson-01-*.md`、`lesson-02-*.md` 等。

## 写作规范（硬约束）

- 公式统一使用 LaTeX 书写。
- 行内公式使用 `$...$`。
- 独立公式使用 `$$...$$`。
- 概念首次出现必须“先解释再使用”，格式统一为：中文术语（English）+ 必要符号。

## SLS 全量执行约束

以下为本课程写作的固定硬约束，必须逐课满足，不得降级为提纲化输出。

### Core 层约束（`sls/10-SLS-Core.md`）

- 动机优先：每个新概念先回答“为什么现在需要它”。
- 结构先于抽象：先建立关系结构，再进入形式化精确化。
- 局部闭环：每课必须形成“问题引入-结构建立-验证-收束”。
- 无知识跳跃：前提、术语、符号必须先定义后使用。
- 塔式依赖：高层结论需可回溯到低层定义。
- 认知回流：形式化结果要回流为可复述直觉。
- 依赖可追溯：结论来源可追溯，避免隐性背景依赖。

### Architecture 层约束（`sls/20-SLS-Architecture.md`）

每课默认覆盖七部件意图（允许轻微合并但不可缺意图）：

- 直觉图像
- 问题意识
- 概念与关系
- 形式化
- 应用与迁移（至少一个完整例子）
- 理解检查（误区纠偏）
- 重直觉化（一句压缩）

### Teaching Flow 层约束（`sls/30-SLS-Teaching-Flow.md`）

- 默认推进：直觉/问题 -> 关系框架 -> 必要形式化 -> 验证 -> 收束 -> 承接。
- 形式化后置为默认；若提前必须说明动机并回嵌解释。
- 节奏控制：避免概念堆叠、避免抽象跃迁无缓冲。
- 章节波形：推进 -> 稳定 -> 压缩 -> 再推进。

### Narrative 层约束（`sls/40-SLS-Narrative-Guide.md`）

- 先问题后定义，先理解后收紧。
- 必须有自然转承，不可 checklist 化压缩。
- 每段要有“当前得到什么”的收束句。
- 结尾必须有“下一步为何必要”的承接句。

### 调用层与任务模式约束（`sls/90/92/93`）

- 完整主题学习采用两阶段：先路径，再逐章展开。
- 展开讲解模式默认开启：不输出骨架替代正文。
- 每章必须有完整例子走完，不可只给目录或定义并列。

## 交付前自检（强制）

每次新增/改写课时前，先自检以下五项：

1. 是否出现先用后讲的概念。
2. 是否缺失本课完整例子。
3. 是否缺失本课闭环与下一课承接。
4. 是否发生结构正确但展开不足的压缩化。
5. 是否存在公式与叙述割裂（只给公式不解释语义）。
