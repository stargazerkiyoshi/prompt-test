# 空间型长期协作系统（第一阶段方向文档）

## 1. 产品概述

本产品旨在构建一个以“空间”为基本形态的长期协作系统，让用户与持续存在的 agent 在其中共同推进长期复杂活动，并通过状态延续、记忆沉淀与行动推进，帮助事情真正做成。

它不是传统聊天机器人，也不是单次任务型 agent 工具。
它更像一个可持续进入的协作环境：用户与 agent 围绕一条长期主线反复进入、持续推进、不断沉淀，最终把复杂事情做成。

第一阶段不追求开放宇宙、多 agent 网络或完整世界系统，而是先做出一个真正有用的**推进空间原型**。

---

## 2. 核心价值

第一阶段的核心价值只有一个：

**帮助用户持续地做成长期复杂活动。**

这里的“长期复杂活动”指的是那些：

* 需要多次进入和反复推进
* 容易中断、遗忘、发散
* 具有阶段性与结构性
* 最终需要产出成果

典型例子包括：

* 系统学习一个主题
* 推进一个长期项目
* 完成一个复杂创作
* 构建一个复杂系统
* 推进一个多阶段个人目标

本产品存在的意义，不是让用户多聊一次，而是让这些事情从“总是中断、总要重启”变成“持续推进、逐步做成”。

---

## 3. 目标用户

第一阶段的目标用户不是泛大众，而是那些长期被复杂活动困住的人。

他们通常具有以下特征：

* 同时推进一个或多个长期目标
* 容易在中途断掉、散掉、忘掉
* 经常和 AI 讨论很多，但难以真正形成持续推进
* 需要的不只是回答，而是长期协作
* 愿意反复进入一个空间继续同一条主线

更具体地说，第一阶段尤其适合以下人群：

### 3.1 长期学习者

想系统学习一个复杂主题，但容易学一阵停一阵。

### 3.2 长期项目推进者

正在推进软件、产品、研究、个人计划等项目，但容易卡住、混乱、失去上下文。

### 3.3 长期创作者 / 建构者

在做写作、世界观、复杂创意系统、长期知识建构时，需要持续沉淀与接续。

---

## 4. 核心场景

第一阶段只聚焦一个总场景：

## 长期复杂活动推进

用户进入系统，不是为了随便聊一聊，而是为了：

* 看清现在在哪
* 明确当前状态
* 决定这次推进什么
* 和 agent 一起完成这一小步
* 留下新的产物和状态
* 让下次进入更容易继续

典型场景包括：

* 学习一门复杂主题
* 推进一个长期项目
* 拆解并完成一个大目标
* 维持一条长期创作主线
* 接续一个复杂想法或系统设计

---

## 5. 用户痛点

第一阶段主要解决以下问题：

### 5.1 容易中断

长期事情很容易断掉，停几天后重新开始成本极高。

### 5.2 容易分散

想法、资料、文档、决策、任务到处都是，缺少统一环境。

### 5.3 容易遗忘

用户会忘记：

* 上次做到哪里
* 为什么做这个决策
* 当前真正卡点是什么
* 下一步原本是什么

### 5.4 容易空转

和 AI 聊了很多、想了很多，但事情没有真正推进。

### 5.5 每次都重新开始

现有很多 AI 交互缺少长期协作连续性，每次都像第一次合作。

---

## 6. 产品定义

第一阶段产品可以定义为：

**一个单空间、单 agent 的长期复杂活动推进系统。**

它服务于一条明确主线，帮助用户在多次进入中持续推进。

### 第一阶段最小形态包括：

* 一个空间
* 一个长期协作 agent
* 一条主线
* 一份当前状态
* 一组推进动作
* 一组可沉淀产物

---

## 7. 为什么是“空间”

“空间”不是视觉装饰，而是产品逻辑。

一个有效空间至少包含三类结构：

### 7.1 规则

这个空间是干什么的，允许什么，不允许什么。

### 7.2 记忆

空间中发生过什么，是否留下来，如何影响后续协作。

### 7.3 关系

用户与 agent 在这个空间中的关系是什么。

因此可以用一个简化公式来理解：

**空间 = 规则 + 记忆 + 关系**

空间的意义在于：
把人与 agent 的互动从“临时对话”变成“可持续进入的活动环境”。

---

## 8. 第一个空间的定位

第一个空间已经明确为：

## 长期复杂活动推进空间

它的重点不是自由闲聊，不是世界沉浸，也不是陪伴感。
它的任务是：

**和用户一起把事情做成。**

这个空间主要解决四个问题：

* 断
* 散
* 忘
* 虚

因此，第一个空间的核心价值不是记录，而是：

**持续推进。**

---

## 9. 第一个空间中的 agent 定位

第一阶段中的 agent，不是泛用 AI，也不是以拟人陪伴为核心的角色。
它首先是一个：

## 推进型长期协作者

它的职责包括：

* 识别当前状态
* 延续历史上下文
* 帮助收束目标
* 帮助拆解下一步
* 在用户卡住时推动前进
* 帮助形成和整理产物
* 维护主线连续性

这里“像人”不是语言风格拟人，而是协作结构上的：

* 连续性
* 记忆性
* 责任性
* 稳定性

---

## 10. 第一阶段核心对象

第一阶段空间可以先围绕四个核心对象构建：

### 10.1 主线

用户当前试图做成的长期复杂活动。

### 10.2 当前状态

主线目前在什么阶段，完成了什么，卡在哪里，下一步建议是什么。

### 10.3 推进行动

用户这次进入空间后，准备如何推进主线。

### 10.4 产物

每次推进后沉淀下来的结构化结果。

这四个对象构成了空间原型的最小骨架。

---

## 11. 设计原则

第一阶段建议遵循以下原则：

### 11.1 一切围绕“继续推进”

任何功能都应服务于“更容易继续”。

### 11.2 先清楚，再丰富

先做清晰的状态和推进逻辑，不急着做复杂空间表现。

### 11.3 先单空间，再多空间

先把一个空间做扎实，再考虑空间谱系。

### 11.4 先单 agent，再多 agent

先把一个长期协作 agent 的体验做出来，再考虑多 agent 生态。

### 11.5 先主线闭环，再世界扩张

先证明“真的能帮助做成事”，再谈更大的空间网络和共创世界。

---

## 12. 范围边界

### 12.1 第一阶段优先做的

* 单空间
* 单 agent
* 单主线或少量主线
* 明确状态结构
* 明确推进动作
* 明确产物沉淀
* 明确本次推进是否成功的判断方式

### 12.2 第一阶段不优先做的

* 无限开放的多空间宇宙
* 多 agent 公共网络
* 外部 agent 大规模进入空间
* 以奇幻世界沉浸为主的体验
* 以陪伴感为主的拟人产品
* 纯展示型 agent 生态模拟
* 所有人和 agent 共创空间的开放平台

这些可以作为未来扩展方向，但不是第一阶段重点。

---

## 13. 成功信号

第一阶段是否成立，不看概念是否酷，而看以下信号：

### 13.1 用户愿意反复进入同一空间

说明空间具备持续性。

### 13.2 用户不再需要每次重新解释背景

说明状态与上下文接续有效。

### 13.3 用户每次进入后都能完成一次明确推进

说明空间不是空转。

### 13.4 用户能看到逐步积累的产物

说明系统具有沉淀能力。

### 13.5 用户感觉“这件事真的在被做成”

说明核心价值成立。

---

## 14. 与“认知 OS”的关系

“认知 OS”更适合作为底层理解框架。

它代表一种对记忆、状态、上下文、方法、工具和执行能力进行组织与调度的环境系统，用于支撑长期复杂活动持续运行。

但在当前阶段，“认知 OS”更适合作为内部架构思想，而不是直接产品名称。

对外更容易成立的表达是：

* 空间型长期协作系统
* 长期复杂活动推进空间
* 面向做成事的协作空间

也就是说：

**认知 OS 是底层理解，推进空间是第一产品形态。**

---

## 15. 后续演进方向

第一阶段成立之后，未来可以逐步扩展为更大的空间系统：

### 15.1 多空间

例如：

* 学习空间
* 项目空间
* 自我探索空间
* 自由对话空间
* 世界空间

### 15.2 多 agent

允许多个 agent 同时进入同一空间，形成协作关系。

### 15.3 共创空间

允许空间、规则、内容和 agent 本身由人和 agent 共同创建。

### 15.4 外部 agent 进入

允许外部 agent 被引入空间，参与协作与活动。

### 15.5 共享空间网络

长期看，系统可从个人空间系统扩展为一个由人和 agent 共同参与的共享智能空间网络。

但这些都建立在第一阶段成立之后。
第一阶段要先证明的是：

**空间型长期协作系统，是否真的能帮助人把事情做成。**

---

## 16. 当前阶段结论

当前方向已经收束为一句清晰的话：

**先做一个帮助用户持续推进长期复杂活动的空间，让用户与一个长期协作 agent 在其中共同维护主线、接续状态、沉淀产物，并逐步把事情做成。**

这就是当前阶段的方向锚点。
