# 空间型长期协作系统

## 产品方向文档（PRD 首页版 / 第一阶段）

## 1. 产品一句话

构建一个面向长期复杂活动的推进空间，让用户与持续存在的 agent 在其中长期协作，通过状态延续、记忆沉淀与行动推进，帮助事情真正做成。

---

## 2. 产品定位

这是一个以“空间”为基本形态的长期协作系统。

它不是传统聊天机器人，不是一次性 agent 自动化工具，也不是单纯的任务管理器。
它更像一个可持续进入的协作环境：用户和一个长期存在的 agent 在其中共同推进一条主线，使复杂活动不再反复中断、散乱和重启。

第一阶段的产品重点，不是构建一个开放宇宙，也不是构建多 agent 生态，而是先做出一个真正有用的**推进空间原型**。

---

## 3. 核心目标

第一阶段的核心目标只有一个：

**帮助用户持续地做成长期复杂活动。**

这里的“做成”不是指完成一个微小任务，而是指：

* 一条长期主线能够持续推进
* 用户不再频繁丢失上下文
* 关键状态可以被接续
* 每次进入都有明确推进
* 过程中会产生可沉淀产物
* 最终活动能够逐步完成

---

## 4. 目标用户

第一阶段的目标用户，不是泛大众，而是那些长期被复杂活动困住的人。

他们通常具有以下特征：

* 同时推进一个或多个长期目标
* 容易在中途断掉、散掉、忘掉
* 经常和 AI 聊很多，但难以真正形成持续推进
* 需要的不只是回答，而是长期协作
* 愿意把一个系统当作“主线空间”反复进入

更具体地说，第一阶段更适合以下用户类型：

### 4.1 长期学习者

想系统学习一个主题，但常常学一阵停一阵，难以持续。

### 4.2 长期项目推进者

在推进软件项目、产品项目、个人计划时，容易卡住、混乱、失去上下文。

### 4.3 长期创作者 / 建构者

在做世界观、写作、研究、复杂系统构建时，需要持续沉淀与接续。

这类用户的共同问题不是“不会开始”，而是**难以持续做成**。

---

## 5. 核心场景

第一阶段不做所有场景，只聚焦一个总场景：

## 长期复杂活动推进

典型使用场景包括：

* 学习一门复杂主题
* 推进一个长期项目
* 拆解并完成一个大目标
* 维持一个长期创作主线
* 接续一个复杂想法或系统设计

用户进入系统，不是为了随便聊一聊，而是为了：

* 看清现在在哪
* 决定这次做什么推进
* 和 agent 一起完成这一小步
* 留下新的产物和状态
* 让下次进入更容易继续

---

## 6. 用户痛点

第一阶段产品要解决的不是所有问题，而是以下几类最核心痛点：

### 6.1 断

长期事情很容易中断。
停几天后，再回来成本极高。

### 6.2 散

想法、资料、文档、决策、任务到处都是，无法形成一个持续环境。

### 6.3 忘

用户会忘记：

* 上次做到哪
* 为什么做这个决策
* 当前真正卡点是什么
* 下一步原本该是什么

### 6.4 虚

和 AI 讨论很多、想法很多，但最终事情并没有真正推进。

### 6.5 每次都重新开始

现在很多 AI 交互都像一次性服务，缺乏长期协作连续性。

---

## 7. 核心价值主张

本产品的核心价值不是“更聪明地回答”，而是：

**让长期复杂活动变得可持续、可接续、可沉淀、可完成。**

用户从中获得的不是一次答案，而是：

* 一个持续存在的主线环境
* 一个记得上下文的长期协作 agent
* 一个每次都能推动一点的推进机制
* 一个不断沉淀成果的空间

---

## 8. 核心产品理念

### 8.1 空间，而不是聊天框

产品的基本单位不是对话，而是空间。
空间意味着目标、规则、历史、关系和状态都能持续存在。

### 8.2 长期协作，而不是临时调用

agent 不只是当下回答问题，而是持续存在于空间中，与用户形成长期协作关系。

### 8.3 推进，而不是陪聊

第一阶段的产品重点不是娱乐和陪伴，而是推进。

### 8.4 沉淀，而不是蒸发

每次进入空间，不应只产生聊天记录，而应尽量形成新的状态和产物。

### 8.5 主线，而不是碎片

系统优先围绕一条条长期主线组织，而不是围绕碎片问题组织。

---

## 9. 第一阶段产品定义

第一阶段产品可以定义为：

**一个单空间、单 agent 的长期复杂活动推进系统。**

它服务于一条明确主线，帮助用户在多次进入中持续推进。

### 第一阶段最小形态包括：

* 一个空间
* 一个长期协作 agent
* 一条主线
* 一份当前状态
* 一组推进动作
* 一组可沉淀产物

---

## 10. 第一阶段中的 agent 定位

第一阶段中的 agent，不是泛用 AI，也不是人格化陪伴对象。
它首先是一个：

**推进型长期协作者**

它的核心职责包括：

* 识别当前状态
* 延续历史上下文
* 帮助拆解下一步
* 在用户卡住时提供推进
* 帮助形成和整理产物
* 维护主线连续性

它不是为了“表现得像人”，而是为了“像一个长期协作者一样可靠”。

---

## 11. 第一阶段中的核心对象

### 11.1 主线

用户当前试图做成的长期复杂活动。

### 11.2 当前状态

主线目前处于什么阶段，完成了什么，卡在哪里，下一步建议是什么。

### 11.3 推进行动

用户这次进入空间，准备采取什么推进动作。

### 11.4 产物

每次推进后沉淀下来的结构化结果。

---

## 12. 设计原则

第一阶段建议遵循以下设计原则：

### 12.1 一切围绕“继续推进”

任何功能都应服务于“更容易继续”。

### 12.2 先清楚，再丰富

先做清晰的状态和推进逻辑，不急着做大量花哨空间表现。

### 12.3 先单空间，再多空间

先把一个空间做扎实，再考虑空间谱系。

### 12.4 先单 agent，再多 agent

先把一个长期协作 agent 的体验做出来，再考虑多 agent 生态。

### 12.5 先主线闭环，再世界扩张

先证明“真的能帮助做成事”，再谈更大的空间网络和共创宇宙。

---

## 13. 非目标

为了防止方向发散，第一阶段明确不做以下内容：

* 不做开放式多 agent 公共社区
* 不做无限空间宇宙
* 不做以奇幻世界沉浸为核心的产品
* 不做以陪伴感为主的拟人 AI 产品
* 不做纯展示型 agent 社会模拟
* 不以“所有人和外部 agent 共创”为第一阶段重点
* 不优先追求复杂世界观包装

这些内容可以作为未来扩展想象，但不属于第一阶段目标。

---

## 14. 第一阶段范围

第一阶段只关心一个问题：

**能不能做出一个让用户和 agent 真正持续推进一条长期主线的空间。**

因此，第一阶段范围建议限制在：

* 单空间
* 单 agent
* 单主线或少量主线
* 明确状态结构
* 明确推进动作
* 明确产物沉淀
* 明确“本次推进是否成功”的判断方式

---

## 15. 成功信号

第一阶段是否成立，不看“概念是否酷”，而看以下信号：

### 15.1 用户愿意反复进入同一空间

说明空间具备持续性。

### 15.2 用户不再需要每次重新解释背景

说明上下文和状态接续有效。

### 15.3 用户每次进入后都能完成一次明确推进

说明空间不是空转。

### 15.4 用户能看到逐步积累的产物

说明系统具有沉淀能力。

### 15.5 用户感觉“这件事真的在被做成”

说明核心价值成立。

---

## 16. 与长期方向的关系

第一阶段不是终点，而是整个系统的起点。

未来可以逐步扩展为：

* 学习空间
* 项目空间
* 自我探索空间
* 自由对话空间
* 世界空间
* 多 agent 协作空间
* 人与 agent 共创空间
* 外部 agent 可进入的共享空间网络

但这些都建立在第一阶段成立之后。
第一阶段要证明的不是“世界观有多大”，而是：

**空间型长期协作系统，是否真的能帮助人把事情做成。**

---

## 17. 当前阶段的一句话收束

**第一阶段先做一个帮助用户持续推进长期复杂活动的空间，让用户与一个长期协作 agent 在其中共同维护主线、接续状态、沉淀产物，并逐步把事情做成。**
